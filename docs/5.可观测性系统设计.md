# GraphAgent 可观测性系统设计

## 1. 背景

三层编排图（GuardAgent → PlanAgent → SubAgent）运行时完全静默。用户输入问题后，CLI 只显示 "Agent 思考中..." 然后长时间等待，期间：

- 不知道当前处于哪个阶段（意图分析？方案制定？子任务执行？）
- 不知道每次调用 LLM 时传入了什么 Prompt、返回了什么结果
- 如果程序卡住（LLM 超时、ReAct 死循环、JSON 解析失败等），无法定位问题

## 2. 设计目标

1. **阶段可见**：实时显示编排流程当前处于哪个阶段，哪个 Agent 在思考
2. **LLM I/O 透明**：展示每次 LLM 交互的 System Prompt、User Message、响应内容、Token 用量
3. **工具调用可追踪**：SubAgent ReAct 循环中的工具调用与返回可见
4. **决策可感知**：GuardAgent 的审核决策（通过/驳回）及理由
5. **可配置**：通过日志级别控制输出详细程度，避免信息过载

## 3. 架构设计

### 3.1 核心思路：LangChain Callback + 手动埋点

| 关注点 | 实现方式 |
|--------|---------|
| LLM 输入/输出 | LangChain `BaseCallbackHandler`，透明拦截所有 `llm.invoke()` |
| 阶段转换 | 在各 node 函数中手动埋点，调用 `OrchestrationTracer.trace_phase()` |
| SubAgent 工具调用 | Callback 的 `on_tool_start` / `on_tool_end` 钩子自动捕获 |
| 输出格式 | 结构化终端输出，双线框 + 缩进 + 标签区分 Agent 角色 |

### 3.2 模块结构

```
src/graph_agent/tracer/
├── __init__.py          # 导出 OrchestrationTracer、get_tracer 等
├── tracer.py            # OrchestrationTracer 主类（单例）
├── llm_callback.py      # LLMCallbackHandler（继承 BaseCallbackHandler）
└── format.py            # 终端格式化工具（分隔线、内容输出）
```

### 3.3 日志级别

通过环境变量 `GRAPHAGENT_LOG_LEVEL` 或 `debug.py --log-level` 控制：

| 级别 | 输出内容 |
|------|---------|
| `off` | 不输出任何追踪信息 |
| `phases` | 仅输出阶段切换和决策信息 |
| `llm_io` | 阶段 + LLM 请求/响应（默认） |
| `full` | 阶段 + LLM I/O + 工具调用详情 |

## 4. 核心组件

### 4.1 OrchestrationTracer

`tracer/tracer.py`

全局单例，管理日志级别和 LLM callback 注册。

```
get_tracer().trace_phase("意图分析", "GuardAgent", "分析用户输入")
get_tracer().trace_decision("GuardAgent", "方案审核通过", "计划完整合理")
get_tracer().trace_error("PlanAgent", "JSON 解析失败")
get_tracer().install()  # 将 LLM callback 注册到 LLMFactory
```

### 4.2 LLMCallbackHandler

`tracer/llm_callback.py`

继承 `langchain_core.callbacks.BaseCallbackHandler`，在 LLM 调用前后自动触发：

- `on_llm_start`：提取并格式化 System Prompt、User Message
- `on_llm_end`：提取响应内容和 Token 用量
- `on_tool_start` / `on_tool_end`：工具调用详情（仅 full 级别）

### 4.3 format.py

`tracer/format.py`

提供终端格式化输出函数，使用 Unicode 双线框和缩进结构：

```
╔══ 阶段: 意图分析 · GuardAgent ═══════════════════╗
║  分析用户输入，判断问题类型与复杂度                ║
╚═══════════════════════════════════════════════════╝

  ┌─ LLM 请求 → GuardAgent ────────────────────────
  │ System: 你是一个意图分析专家...
  │ User: 帮我写一个Python脚本
  └────────────────────────────────────────────────

  ┌─ LLM 响应 ← GuardAgent · 234 tokens ───────────
  │ {"intent": "编写Python脚本", "complexity": 2}
  └────────────────────────────────────────────────

  ✓ 方案审核通过 — 计划完整合理，可行性强
```

输出指向 `sys.stderr`，不影响正常 `stdout` 输出。

## 5. 集成方式

### 5.1 LLMFactory 扩展

`llm/base.py` 中 `LLMFactory` 增加：

```python
_callbacks: ClassVar[list] = []

@classmethod
def register_callback(cls, callback) -> None:
    if callback not in cls._callbacks:
        cls._callbacks.append(callback)

@classmethod
def get_callbacks(cls) -> list:
    return list(cls._callbacks)
```

### 5.2 Provider 注入

每个 Provider 的 `get_chat_model()` 从 `LLMFactory.get_callbacks()` 获取 callback 列表，传入模型构造函数：

```python
callbacks = LLMFactory.get_callbacks()
return ChatOpenAI(
    model=..., api_key=..., ...,
    callbacks=callbacks if callbacks else None,
)
```

所有 LLM 调用自动被 callback 拦截，无需修改调用方代码。

### 5.3 Node 埋点位置

| 文件 | 函数 | 埋点内容 |
|------|------|---------|
| `node/guard.py` | `_analyze_intent()` | `trace_phase("意图分析", "GuardAgent")` |
| `node/guard.py` | `_review_plan()` | `trace_phase("方案审核", "GuardAgent")` + `trace_decision()` |
| `node/guard.py` | `_review_result()` | `trace_phase("结果验收", "GuardAgent")` + `trace_decision()` |
| `node/plan.py` | `_generate_plan()` | `trace_phase("方案制定", "PlanAgent")` |
| `node/plan.py` | `_dispatch_tasks()` | `trace_phase("任务派发", "PlanAgent")` |
| `node/plan.py` | `_synthesize_results()` | `trace_phase("结果汇总", "PlanAgent")` |
| `node/subagent.py` | `subagent_exec_node()` | `trace_phase("子任务执行", "SubAgent")` |
| `node/subagent.py` | `_run_subagent()` | `trace_phase("执行子任务 [id]", config.name)` |

### 5.4 debug.py 初始化

```python
from graph_agent.tracer import OrchestrationTracer

tracer = OrchestrationTracer()  # 从环境变量读取日志级别
tracer.install()                # 注册 LLM callback
```

## 8. 使用方式

```bash
# 默认：llm_io 级别
py debug.py

# 最详细：包含工具调用
py debug.py --log-level full

# 仅阶段切换
py debug.py --log-level phases

# 关闭追踪
py debug.py --log-level off

# 通过环境变量
set GRAPHAGENT_LOG_LEVEL=full && py debug.py
```
