# GraphAgent 记忆系统设计文档

## 1. 概述

为了支撑多轮对话、多轮流程编排、工具调用上下文复用以及支持长期自主任务、反思迭代，
GraphAgent 需要具备对聊天记录和任务进度等信息进行记忆的能力。
此外，由于 LLM 上下文窗口的限制，GraphAgent 还需具备对不重要记忆进行压缩和遗忘的能力。

记忆系统是 GraphAgent 的核心基础设施之一，它使得 Agent 能够：
- 在多轮对话中保持上下文连贯性
- 在长时间运行的任务中追踪进度和中间状态
- 复用历史工具调用结果，减少重复计算
- 通过反思迭代从历史经验中学习和改进

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| **多维记忆** | 支持对话记忆、任务记忆、工具上下文记忆、知识记忆等多种记忆类型 |
| **分层存储** | 短期记忆（详情）+ 长期记忆（压缩），平衡精度与容量 |
| **智能检索** | 基于相关性和时效性的记忆检索，支持语义搜索 |
| **自动压缩** | 对旧记忆自动生成摘要，释放上下文窗口 |
| **动态遗忘** | 基于重要性、时效性、容量上限的自动遗忘机制 |
| **上下文复用** | 工具调用结果可缓存复用，减少 LLM 调用成本 |
| **反思迭代** | 支持对历史记忆的总结、提炼和高阶洞察生成 |
| **持久化** | 记忆可跨会话持久化，支持长期自主任务 |
| **LangGraph 兼容** | 与现有 LangGraph StateGraph 架构无缝集成 |

## 4. 记忆分类体系

### 4.1 记忆类型（MemoryType）

```python
from enum import Enum

class MemoryType(Enum):
    CONVERSATION = "conversation"   # 对话记录：用户与 Agent 的交互历史
    TASK = "task"                   # 任务进度：当前任务的分解、状态和中间结果
    TOOL_CONTEXT = "tool_context"   # 工具上下文：工具调用的输入输出缓存
    KNOWLEDGE = "knowledge"         # 知识记忆：从交互中提取的事实、规则和模式
    REFLECTION = "reflection"       # 反思记录：对历史行为的总结和评估
```

| 类型 | 内容示例 | 生命周期 | 压缩策略 |
|------|---------|---------|---------|
| CONVERSATION | 用户提问、Agent 回复、中间推理过程 | 单会话为主，重要内容可长期保留 | 近 N 轮保留详情，旧轮生成摘要 |
| TASK | 任务分解树、子任务状态、进度百分比 | 任务周期内活跃，完成后归档 | 进行中详情保留，已完成生成摘要 |
| TOOL_CONTEXT | 工具名、输入参数、返回结果、耗时 | 短期缓存，按 TTL 过期 | 相同调用覆盖，不同结果追加 |
| KNOWLEDGE | 用户偏好、领域事实、推理规则 | 长期积累，跨会话持久 | 合并同类知识，去重和更新 |
| REFLECTION | 任务完成后的复盘、失败分析、改进建议 | 长期保留，供后续任务参考 | 合并相似反思，按领域归档 |

### 4.2 记忆条目（MemoryEntry）

```python
@dataclass
class MemoryEntry:
    """单条记忆记录。"""

    id: str                          # 唯一标识
    type: MemoryType                 # 记忆类型
    content: str                     # 记忆内容
    importance: float                # 重要性评分 (0.0 ~ 1.0)
    created_at: float                # 创建时间戳
    last_accessed_at: float          # 最后访问时间
    access_count: int                # 访问次数
    ttl: Optional[float]             # 过期时间（秒），None 表示永不过期
    metadata: dict[str, Any]         # 额外元数据（来源、关联任务ID等）
    embedding: Optional[list[float]] # 内容向量（用于语义检索）
    compressed_from: Optional[list[str]]  # 从哪些记忆条目压缩而来
```

### 5.2 短期记忆存储（ShortTermStore）

- **存储介质**：内存（Python dict/list）
- **容量限制**：可配置（默认最多保留 N 条记忆，或总 token 数上限）
- **记忆层级**：L0（原始详情）
- **特点**：读写快速，支持即时访问，会话结束可丢弃

```python
class ShortTermStore:
    """短期记忆存储。"""

    def __init__(self, max_entries: int = 1000, max_tokens: int = 100000):
        self.max_entries = max_entries
        self.max_tokens = max_tokens
        self._entries: dict[str, MemoryEntry] = {}

    def add(self, entry: MemoryEntry) -> None: ...
    def get(self, entry_id: str) -> Optional[MemoryEntry]: ...
    def query(
        self,
        mem_type: Optional[MemoryType] = None,
        limit: int = 50,
        recency_weight: float = 0.5,
    ) -> list[MemoryEntry]: ...
    def remove(self, entry_id: str) -> None: ...
    def clear(self) -> None: ...
    def token_count(self) -> int: ...
```

### 5.3 长期记忆存储（LongTermStore）

- **存储介质**：SQLite 数据库或 JSON 文件
- **记忆层级**：L1（摘要）、L2（关键点/规则）
- **特点**：持久化存储，支持跨会话恢复，容量可扩展

```python
class LongTermStore:
    """长期记忆存储。"""

    def __init__(self, db_path: str = "~/.graph_agent/memory.db"):
        self.db_path = db_path

    def _init_schema(self) -> None: ...
    def save(self, entry: MemoryEntry) -> None: ...
    def load(self, entry_id: str) -> Optional[MemoryEntry]: ...
    def query(
        self,
        mem_type: Optional[MemoryType] = None,
        keyword: Optional[str] = None,
        time_range: Optional[tuple[float, float]] = None,
        importance_threshold: float = 0.0,
        limit: int = 50,
    ) -> list[MemoryEntry]: ...
    def delete(self, entry_id: str) -> None: ...
    def get_stats(self) -> dict[str, Any]: ...
```

### 6.2 CompressionEngine（压缩引擎）

负责对旧记忆进行摘要、合并和层级压缩，核心策略采用 **三级记忆金字塔**：

```
            ┌─────────┐
            │  L2     │  ← 关键规则/洞察（极少，永久保留）
            │  规则层  │
        ┌───┴─────────┴───┐
        │      L1          │  ← 摘要层（中等数量，长期存储）
        │     摘要层        │
    ┌───┴──────────────────┴───┐
    │           L0              │  ← 详情层（大量，短期存储）
    │          详情层            │
    └──────────────────────────┘
```

| 层级 | 内容 | 存储位置 | 压缩触发条件 |
|------|------|---------|------------|
| L0 | 原始对话、完整工具调用结果 | ShortTermStore | 超过 N 条或超过 M 分钟后触发 |
| L1 | 对话摘要、任务阶段总结 | LongTermStore | 同类 L1 超过阈值时合并 |
| L2 | 规则、偏好、经验教训 | LongTermStore | 手动触发或重要洞察生成 |

```python
class CompressionEngine:
    """记忆压缩引擎。"""

    def __init__(self, llm_provider: "LLMProvider"):
        self.llm = llm_provider

    def compress_to_summary(self, entries: list[MemoryEntry]) -> MemoryEntry:
        """将多条 L0 记忆压缩为一条 L1 摘要。"""
        ...

    def merge_summaries(self, entries: list[MemoryEntry]) -> MemoryEntry:
        """合并多条 L1 摘要。"""
        ...

    def extract_insight(self, entries: list[MemoryEntry]) -> MemoryEntry:
        """从记忆中提取 L2 级别的规则或洞察。"""
        ...

    def evaluate_compressibility(self, entry: MemoryEntry) -> float:
        """评估一条记忆的可压缩程度。"""
        ...
```

### 6.3 ForgettingEngine（遗忘引擎）

负责清理不重要的记忆以释放容量，采用多维度评分机制。

**遗忘策略：**

| 策略 | 说明 | 权重 |
|------|------|------|
| **重要性衰减** | 记忆的重要性随访问频率和时间衰减 | 0.35 |
| **时效性评估** | 超过 TTL 或长时间未访问的记忆优先遗忘 | 0.30 |
| **容量压力** | 当存储接近容量上限时加速遗忘 | 0.20 |
| **冗余检测** | 内容相似度高的记忆保留最新/最重要的 | 0.15 |

```python
class ForgettingEngine:
    """记忆遗忘引擎。"""

    def __init__(
        self,
        importance_decay_rate: float = 0.01,   # 重要性衰减率（每小时）
        default_importance: float = 0.5,         # 默认重要性
        max_capacity: int = 10000,               # 最大记忆条目数
    ):
        self.importance_decay_rate = importance_decay_rate
        self.default_importance = default_importance
        self.max_capacity = max_capacity

    def evaluate_importance(self, entry: MemoryEntry) -> float:
        """综合评估记忆的当前重要性。"""
        ...

    def calculate_retention_score(self, entry: MemoryEntry) -> float:
        """计算记忆的保留评分。评分越低，越优先被遗忘。"""
        ...

    def select_victims(self, entries: list[MemoryEntry], count: int) -> list[MemoryEntry]:
        """选择需要遗忘的记忆条目。"""
        ...

    def should_forget(self, entry: MemoryEntry) -> bool:
        """判断单条记忆是否应该被遗忘。"""
        ...

    def garbage_collect(
        self,
        short_term: ShortTermStore,
        long_term: LongTermStore,
        target_ratio: float = 0.8,
    ) -> int:
        """执行垃圾回收，使存储占用降到目标比例。"""
        ...
```

**保留评分公式：**

```python
def calculate_retention_score(entry: MemoryEntry) -> float:
    age_hours = (now() - entry.created_at) / 3600
    idle_hours = (now() - entry.last_accessed_at) / 3600

    # 重要性：基础分 + 访问加成，随时间衰减
    importance = entry.importance * (1 + log(1 + entry.access_count))
    importance *= exp(-importance_decay_rate * idle_hours)

    # 时效性：近期记忆更保值
    recency = exp(-0.01 * age_hours)

    # 关联性：被关联越多越保值
    connectivity = min(1.0, len(entry.metadata.get("linked_from", [])) / 10)

    return (
        0.40 * importance +
        0.30 * recency +
        0.15 * connectivity +
        0.15 * (1.0 if entry.ttl is None else max(0, (entry.ttl - age_hours) / entry.ttl))
    )
```

## 7. 与 LangGraph 集成

### 7.1 扩展 State 定义

```python
from langgraph.graph import StateGraph, MessagesState
from typing import TypedDict, Optional

class GraphAgentState(TypedDict):
    """GraphAgent 的状态定义，包含记忆字段。"""
    messages: list          # LangGraph 消息列表
    memory: dict            # MemoryManager 序列化状态
    current_task_id: Optional[str]   # 当前执行的任务 ID
    context_summary: str    # 当前上下文摘要（注入 LLM）
```

### 7.2 记忆感知节点

```python
# 在 graph.py 中集成记忆节点

from graph_agent.memory import MemoryManager

def remember_node(state: GraphAgentState) -> dict:
    """记忆节点：将当前交互写入记忆。"""
    memory_manager = MemoryManager.from_state(state["memory"])
    last_message = state["messages"][-1]
    memory_manager.remember_conversation(
        role=last_message.type,
        content=last_message.content,
    )
    return {"memory": memory_manager.save_state()}

def recall_node(state: GraphAgentState) -> dict:
    """回忆节点：检索相关记忆并注入上下文。"""
    memory_manager = MemoryManager.from_state(state["memory"])
    context = memory_manager.get_context_for_llm(max_tokens=6000)
    return {"context_summary": context}

def reflect_node(state: GraphAgentState) -> dict:
    """反思节点：在任务完成后进行复盘。"""
    memory_manager = MemoryManager.from_state(state["memory"])
    task_entries = memory_manager.recall_task_progress(state["current_task_id"])
    reflection = memory_manager.reflect(task_entries)
    return {"memory": memory_manager.save_state()}

def compress_node(state: GraphAgentState) -> dict:
    """压缩节点：定期触发记忆压缩和遗忘。"""
    memory_manager = MemoryManager.from_state(state["memory"])
    compressed_count = memory_manager.compress()
    forgotten_count = memory_manager.forget()
    return {"memory": memory_manager.save_state()}
```

## 8. 上下文窗口管理

### 8.1 Token 预算分配

LLM 上下文窗口有限，需合理分配 token 预算：

| 组件 | 默认占比 | 说明 |
|------|---------|------|
| 系统提示词 | ~10% | 角色设定、行为约束 |
| 工具定义 | ~15% | 所有可用工具的 schema |
| 近期对话详情（L0） | ~35% | 最近 N 轮对话的完整内容 |
| 历史摘要（L1） | ~25% | 更早对话的压缩摘要 |
| 知识/规则（L2） | ~10% | 提取的关键规则和偏好 |
| 安全余量 | ~5% | 防止溢出 |

### 8.2 动态上下文组装

```python
def build_context_window(
    self,
    system_prompt: str,
    tools_schema: str,
    max_tokens: int = 8000,
) -> str:
    """动态组装上下文窗口。"""
    budget = TokenBudget(max_tokens)

    # 各组件按优先级分配 token
    context_parts = []

    # 1. 系统提示词（固定）
    budget.allocate("system", len(system_prompt))
    context_parts.append(system_prompt)

    # 2. 工具定义（固定）
    budget.allocate("tools", len(tools_schema))
    context_parts.append(tools_schema)

    # 3. L2 规则/偏好（最高优先级动态内容）
    l2_memories = self.long_term.query(
        mem_type=[MemoryType.KNOWLEDGE, MemoryType.REFLECTION],
        importance_threshold=0.7,
    )
    l2_text = self._format_memories(l2_memories)
    budget.allocate("L2_knowledge", len(l2_text))
    context_parts.append(l2_text)

    # 4. L1 历史摘要
    l1_remaining = budget.remaining * 0.45
    l1_memories = self.short_term.query(mem_type=MemoryType.CONVERSATION)
    l1_text = self._format_memories(l1_memories, max_chars=l1_remaining)
    context_parts.append(l1_text)

    # 5. L0 近期对话详情
    l0_remaining = budget.remaining
    l0_memories = self.short_term.query(limit=20)
    l0_text = self._format_memories(l0_memories, max_chars=l0_remaining)
    context_parts.append(l0_text)

    return "\n\n".join(context_parts)
```

from graph_agent.memory import MemoryManager, create_memory_manager

memory_manager = create_memory_manager()

def agent_node(state: GraphAgentState) -> dict:

    }

## 11. 设计优势

1. **分层解耦**：记忆存储、压缩、遗忘、检索各司其职，可独立替换和优化
2. **三级金字塔**：L0（详情）→ L1（摘要）→ L2（规则），随层级升高信息密度增大
3. **智能遗忘**：多维度评分机制确保重要信息不被误删，非关键信息及时释放
4. **LangGraph 原生集成**：通过 StateGraph 的自定义 State 实现与图执行引擎的无缝结合
5. **可配置**：所有策略参数（容量、阈值、衰减率）均通过环境变量控制
6. **可扩展**：存储后端可替换（如升级为向量数据库），压缩/遗忘策略可插拔
7. **跨会话持久化**：长期记忆支持跨会话恢复，支撑持续数天的自主任务
8. **上下文复用**：工具调用结果缓存减少重复 LLM 调用，降低成本和延迟
