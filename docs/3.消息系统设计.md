# GraphAgent 消息系统设计文档

## 1. 概述

为了统一不同 Agent 之间通信时使用的消息格式，方便实现后续的 Agent 编排系统，GraphAgent 实现了统一的消息模块。
该模块在 LangGraph 原生的 `HumanMessage` / `AIMessage` / `ToolMessage` / `SystemMessage` 之上提供更高阶的抽象，
屏蔽底层消息类型的差异，向上层业务暴露统一的 `MessageBlock` 格式。后续 ReAct 循环中将使用自定义的 `MessageBlock` 
而非 LangGraph 自带的 `BaseMessage` 进行消息传递。

## 2. 目录结构

```
src/graph_agent/
├── message/
│   ├── __init__.py          # 公共接口导出
│   ├── base.py              # 核心基础类（ToolUseBlock/ToolResultBlock/ContentBlock/MessageBlock）
│   ├── convert.py           # MessageBlock 与 LangChain BaseMessage 之间的双向转换
│   ├── utils.py             # 消息通用工具（消息ID生成、工厂函数、消息校验等）
│   └── message_type.py      # 详细定义 MessageType 的所有枚举值
└── graph.py                 # Agent 图定义（从 message 模块导入）
```

## 3. 核心组件设计

### 3.1 MessageBlock — 统一消息体

`MessageBlock` 是消息系统的核心数据结构，所有 Agent 间通信均使用该类型。它屏蔽了底层 LangChain 多种消息类型的差异，提供统一、可扩展的消息抽象。

```python
@dataclass
class MessageBlock:
    role: Literal["user", "assistant", "system", "tool"]
    content: str | list[ContentBlock]
    name: str
    message_type: str
    message_id: str
    metadata: dict[str, Any] = None
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `role` | Literal["user", "assistant", "system", "tool"] | 消息来源角色，对应对话中的参与者 |
| `content` | str \| list[ContentBlock] | 消息内容。简单文本时使用 `str`，需承载多模态/工具调用等结构化信息时使用 `list[ContentBlock]` |
| `name` | str | 对 role 的进一步细化。当 role 为 `"tool"` 时 name 为工具名称；当 role 为 `"assistant"` 时 name 为 Agent 名称；当 role 为 `"user"` 时可为空字符串 |
| `message_type` | str | 消息类型标识，由 `MessageType` 枚举定义，精确描述此消息的语义类别 |
| `message_id` | str | 消息的唯一标识符，由 `generate_message_id()` 生成 |
| `metadata` | dict[str, Any] | 扩展字段，承载任意键值对的元信息（如时间戳、token 用量、来源节点等） |

**role 与 message_type 的对应关系：**

| role | 可选 message_type | 说明 |
|------|-------------------|------|
| `user` | `user_input`, `user_interrupt`, `user_feedback` | 用户侧消息 |
| `assistant` | `agent_response`, `agent_think`, `agent_action`, `agent_plan`, `agent_summary`, `agent_reflection`, `agent_error` | Agent 侧消息 |
| `system` | `system_prompt`, `system_notification`, `system_state` | 系统级消息 |
| `tool` | `tool_result` | 工具执行结果 |

### 3.2 ContentBlock — 结构化内容块

当消息内容不能简单地用纯文本表示时（如工具调用、工具结果、思考过程），使用 `ContentBlock` 列表来精确描述每种内容片段。

```python
@dataclass
class ContentBlock:
    block_type: Literal["text", "tool_use", "tool_result", "thinking"]
    text: str | None = None
    tool_use: ToolUseBlock | None = None
    tool_result: ToolResultBlock | None = None
    thinking: str | None = None
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `block_type` | Literal["text", "tool_use", "tool_result", "thinking"] | 内容块类型 |
| `text` | str \| None | `block_type="text"` 时使用，存储纯文本 |
| `tool_use` | ToolUseBlock \| None | `block_type="tool_use"` 时使用，描述工具调用请求 |
| `tool_result` | ToolResultBlock \| None | `block_type="tool_result"` 时使用，描述工具调用结果 |
| `thinking` | str \| None | `block_type="thinking"` 时使用，存储 Agent 的思考过程 |

**block_type 使用场景：**

| block_type | 携带字段 | 典型场景 |
|------------|----------|----------|
| `text` | `text` | 普通文本回复、用户输入 |
| `tool_use` | `tool_use` | Agent 决定调用某个工具 |
| `tool_result` | `tool_result` | 工具执行完毕返回结果 |
| `thinking` | `thinking` | Agent 内部推理过程（支持"思维链"展示） |

### 3.3 ToolUseBlock — 工具调用块

描述一次工具调用的完整信息。

```python
@dataclass
class ToolUseBlock:
    tool_id: str
    tool_name: str
    input_args: dict[str, Any]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `tool_id` | str | 工具调用唯一标识，用于关联 ToolResultBlock |
| `tool_name` | str | 被调用的工具名称 |
| `input_args` | dict[str, Any] | 传递给工具的参数键值对 |

### 3.4 ToolResultBlock — 工具结果块

描述一次工具调用的执行结果。

```python
@dataclass
class ToolResultBlock:
    tool_id: str
    tool_name: str
    output: str
    status: Literal["success", "failure"]
    error_message: str | None = None
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `tool_id` | str | 对应的工具调用 ID，与 ToolUseBlock.tool_id 匹配 |
| `tool_name` | str | 被调用的工具名称 |
| `output` | str | 工具执行返回的输出内容 |
| `status` | Literal["success", "failure"] | 执行状态：成功或失败 |
| `error_message` | str \| None | 当 status 为 `"failure"` 时记录错误详情 |

## 4. MessageType 枚举设计

`message_type.py` 中定义 `MessageType` 枚举，统一管理所有消息类型标识，避免字符串硬编码。

```python
from enum import Enum

class MessageType(Enum):
    # === 用户侧 ===
    USER_INPUT = "user_input"           # 用户输入
    USER_INTERRUPT = "user_interrupt"   # 用户中断当前执行
    USER_FEEDBACK = "user_feedback"     # 用户对 Agent 输出的反馈

    # === Agent 侧 ===
    AGENT_RESPONSE = "agent_response"   # Agent 最终文本回复
    AGENT_THINK = "agent_think"         # Agent 内部推理 / 思维链
    AGENT_ACTION = "agent_action"       # Agent 决定调用工具
    AGENT_PLAN = "agent_plan"           # Agent 生成的执行计划
    AGENT_SUMMARY = "agent_summary"     # Agent 对上下文的摘要压缩
    AGENT_REFLECTION = "agent_reflection"  # Agent 自反思 / 自评估
    AGENT_ERROR = "agent_error"         # Agent 自身错误

    # === 系统侧 ===
    SYSTEM_PROMPT = "system_prompt"         # 系统提示词
    SYSTEM_NOTIFICATION = "system_notification"  # 系统通知（超时、限额等）
    SYSTEM_STATE = "system_state"           # 状态变更通知

    # === 工具侧 ===
    TOOL_RESULT = "tool_result"         # 工具执行结果
```

## 5. 消息转换层设计

`convert.py` 负责在 `MessageBlock` 与 LangChain `BaseMessage` 之间进行双向转换。桥接层使得上层编排逻辑只需操作 `MessageBlock`，而在与 LangGraph 运行时交互时自动完成格式转换。

### 5.1 转换映射关系

| LangChain 类型 | MessageBlock |
|---------------|-------------|
| `HumanMessage` | `role="user"`, `message_type="user_input"` |
| `AIMessage`（含 tool_calls） | `role="assistant"`, `message_type="agent_action"`, `content` 包含 `ToolUseBlock` 列表 |
| `AIMessage`（不含 tool_calls） | `role="assistant"`, `message_type="agent_response"` |
| `ToolMessage` | `role="tool"`, `message_type="tool_result"`, `name` 为工具名 |
| `SystemMessage` | `role="system"`, `message_type="system_prompt"` |

### 5.2 核心转换函数

```python
def agent_message_to_langchain(msg: MessageBlock) -> BaseMessage:
    """将单个 MessageBlock 转换为 LangChain BaseMessage 子类实例。"""

def langchain_to_agent_message(msg: BaseMessage) -> MessageBlock:
    """将单个 LangChain BaseMessage 转换为 MessageBlock。"""

def agent_messages_to_langchain(msgs: list[MessageBlock]) -> list[BaseMessage]:
    """批量将 MessageBlock 列表转换为 LangChain BaseMessage 列表。"""

def langchain_to_agent_messages(msgs: list[BaseMessage]) -> list[MessageBlock]:
    """批量将 LangChain BaseMessage 列表转换为 MessageBlock 列表。"""
```

### 5.3 转换细节说明

**AIMessage 含 tool_calls 时的处理：**
LangChain 的 `AIMessage` 通过 `tool_calls` 属性携带工具调用信息。转换时需将每个 `tool_call` 映射为一个 `ContentBlock(block_type="tool_use", tool_use=ToolUseBlock(...))`，最终的 `MessageBlock.content` 为这些 ContentBlock 组成的列表。若 AIMessage 同时包含文本内容和 tool_calls，则文本部分作为 `ContentBlock(block_type="text")` 置于列表首位。

**ToolMessage 转换：**
LangChain 的 `ToolMessage` 携带 `tool_call_id` 和 `name`。转换后 `MessageBlock.role="tool"`，`MessageBlock.name` 取 `ToolMessage.name`，`MessageBlock.message_type=MessageType.TOOL_RESULT`，`content` 为 `ToolMessage.content` 字符串。

**不可识别类型的降级策略：**
遇到无法精确映射的 LangChain 消息类型时，将其转为 `role="system"`、`message_type="system_notification"` 的 MessageBlock，并在 `metadata` 中保留原始类型信息，确保不丢失数据。

## 7. 使用示例

### 7.1 在 ReAct 循环中使用 MessageBlock

```python
from graph_agent.message import (
    MessageBlock, ContentBlock, ToolUseBlock,
    create_user_message, create_assistant_message, create_tool_result_message,
)
from graph_agent.message.message_type import MessageType

# 用户输入
user_msg = create_user_message(content="帮我查一下北京今天天气")

# Agent 思考（思维链）
think_msg = create_assistant_message(
    content=[ContentBlock(block_type="thinking", thinking="用户想查天气，需要调用天气工具")],
    message_type=MessageType.AGENT_THINK,
)

# Agent 决定调用工具
action_msg = create_assistant_message(
    content=[
        ContentBlock(
            block_type="tool_use",
            tool_use=ToolUseBlock(
                tool_id="call_001",
                tool_name="get_weather",
                input_args={"city": "北京"}
            )
        )
    ],
    message_type=MessageType.AGENT_ACTION,
)

# 工具返回结果
tool_msg = create_tool_result_message(
    tool_id="call_001",
    tool_name="get_weather",
    output="北京今天晴，22°C",
)

# Agent 生成最终回复
response_msg = create_assistant_message(
    content="北京今天天气晴朗，气温22°C，适合出行。",
    message_type=MessageType.AGENT_RESPONSE,
)
```

### 7.2 与 LangGraph 集成时的消息转换

```python
from graph_agent.message.convert import (
    agent_messages_to_langchain,
    langchain_to_agent_messages,
)

# 将内部 MessageBlock 列表转为 LangChain 格式后传给 LLM
langchain_msgs = agent_messages_to_langchain(agent_messages)
llm_response = llm.invoke(langchain_msgs)

# 将 LLM 返回的 LangChain 消息转回 MessageBlock
agent_msg = langchain_to_agent_message(llm_response)
```

### 7.3 与现有 graph.py 集成的演进方向

当前 `graph.py` 使用 `MessagesState` 和 LangChain 原生消息。引入消息模块后，图节点内部使用 `MessageBlock` 进行推理，仅在调用 LLM 和工具节点时通过 `convert` 模块完成格式转换，使编排逻辑与底层消息格式解耦。

## 8. 设计优势

1. **统一抽象** — 将 LangChain 的 4 种 BaseMessage 子类统一为一个 `MessageBlock`，降低上层编排逻辑的复杂度
2. **结构化内容** — 通过 `ContentBlock` 支持文本、工具调用、工具结果、思考过程等多种内容类型在同一消息中共存
3. **可扩展** — `metadata` 字段和 `MessageType` 枚举支持按需扩展，无需修改核心数据结构
4. **双向兼容** — 通过 `convert` 模块保持与 LangChain/LangGraph 生态的完全兼容
5. **语义清晰** — `message_type` 比 LangChain 原生的消息类型提供更丰富的语义，直接表达消息在 Agent 协作中的含义
6. **可追溯** — 每条消息都有全局唯一 ID，便于调试、日志追踪和消息关联
7. **解耦底层** — 上层编排逻辑不依赖 LangChain 具体类型，更换底层框架时只需修改 convert 层
