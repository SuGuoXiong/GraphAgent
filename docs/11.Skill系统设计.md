# GraphAgent Skill 系统设计文档

## 1. 概述

### 1.1 设计背景

当前 GraphAgent 的 SubAgent 系统通过 `config/subagents.yaml` 手动配置，存在以下局限：

- **配置分散**：SubAgent 的元数据（名称、描述、技能标签）在 YAML 中，提示词在 `prompts/` 目录下，两者分离管理，新增一个 SubAgent 需要在多处操作
- **能力描述不够丰富**：YAML 中的 `skills` 仅为简单的字符串标签，PlanAgent 只能看到一个技能名称，无法理解该技能的详细执行步骤、适用场景和限制条件
- **缺乏自包含性**：一个完整的"技能"（如 PDF 解析）除了提示词外，往往还需要示例文档（reference）和辅助脚本（scripts），现有结构无法将这三者组织在一起
- **扩展门槛高**：对于复杂技能，用户需要同时理解 YAML 配置格式、提示词文件路径约定、工具注册机制等多个概念

Skill 系统将技能定义从"配置驱动"升级为"文档驱动"——每个 Skill 是一个自包含的文件夹，内含功能说明、执行步骤、参考示例和辅助脚本。系统在启动时自动扫描并注册所有 Skill，根据 SKILL.md 中的描述自动生成对应的 SubAgent。

### 1.2 核心设计理念

```
现有方式（配置驱动）:
  config/subagents.yaml  →  手写 YAML 配置  →  SubAgent
  prompts/subagent/*.txt →  手写提示词     →  系统提示词

Skill 方式（文档驱动）:
  skills/{skill_name}/SKILL.md      →  技能定义文档  →  自动生成 SubAgent 配置 + 系统提示词
  skills/{skill_name}/reference/   →  参考示例     →  辅助 LLM 理解
  skills/{skill_name}/scripts/     →  辅助脚本     →  执行时调用
```

### 1.3 与现有系统的关系

Skill 系统**增强**而非**替代**现有 SubAgent 系统：

- `config/subagents.yaml` 中手动定义的 SubAgent 继续有效
- Skill 系统扫描 `skills/` 目录，自动为每个 Skill 生成对应的 SubAgent 配置
- 两套来源的 SubAgent 统一注册到 `SubAgentRegistry` 中，PlanAgent 无差别调度
- 当 Skill 定义的 SubAgent 与 YAML 中的 SubAgent 同名时，Skill 定义优先（覆盖）

### 2.2 SKILL.md 文件规范

SKILL.md 是 Skill 的核心定义文件，采用 Markdown 格式，包含标准的 frontmatter（YAML 头部）和正文内容。

#### 2.2.1 完整格式

```markdown
---
name: file-management
description: 文件系统读写与管理，支持文本文件的读取、写入、追加、删除，目录遍历，文件存在性检查，以及批量文件操作
tools:
  - name: read_file
    description: 读取指定路径的文件内容，支持 UTF-8 / GBK 编码
    parameters:
      - name: path
        type: string
        description: 文件绝对路径
        required: true
      - name: encoding
        type: string
        description: 文件编码格式，默认 utf-8
        required: false
  - name: write_file
    description: 将内容写入指定文件，如文件不存在则创建
    parameters:
      - name: path
        type: string
        description: 文件绝对路径
        required: true
      - name: content
        type: string
        description: 要写入的内容
        required: true
max_iterations: 3
llm_config:
  model: null
  temperature: 0.1
---

# 文件管理技能

## 功能概述
对本地文件系统进行安全的读写操作，支持文本文件的常见操作和目录管理。

## 适用场景
- 需要读取文件内容并提取信息
- 需要将生成的结果保存到文件
- 需要列出目录结构或检查文件是否存在
- 需要批量处理多个文件

## 执行步骤

### 场景一：读取并分析文件内容
1. 使用 `read_file` 工具读取目标文件内容
2. 分析文件内容，提取用户需要的信息
3. 如果文件编码异常，尝试不同编码重新读取
4. 将分析结果返回

### 场景二：写入文件
1. 确认目标目录是否存在，必要时使用 `list_directory` 检查
2. 使用 `write_file` 工具将内容写入指定路径
3. 使用 `read_file` 工具验证写入结果
4. 返回写入结果确认

### 场景三：批量文件操作
1. 使用 `list_directory` 获取目标目录下的文件列表
2. 根据用户需求筛选目标文件
3. 逐个处理文件（读取/修改/删除）
4. 汇总处理结果并返回

## 限制条件
- 不会删除包含重要系统文件的目录
- 写入前会确认目标路径的安全性和合法性
- 大文件（>10MB）读取时会提示用户确认
```

#### 2.2.2 Frontmatter 字段说明

| 字段 | 必需 | 类型 | 说明 |
|------|------|------|------|
| `name` | 是 | `string` | Skill 唯一标识名称，对应 SubAgent 名称（如 `file-management`） |
| `description` | 是 | `string` | Skill 功能的一句话描述，供 PlanAgent 理解和匹配 |
| `tools` | 否 | `list[ToolDef]` | 该 Skill 需要使用的工具列表。如不需要工具（纯 LLM 推理），可省略 |
| `tools[].name` | 是 | `string` | 工具名称，对应 `ToolCenter` 中注册的工具 |
| `tools[].description` | 是 | `string` | 工具的功能描述，用于生成 SubAgent 系统提示词 |
| `tools[].parameters` | 是 | `list[ParamDef]` | 工具的参数列表 |
| `parameters[].name` | 是 | `string` | 参数名称 |
| `parameters[].type` | 是 | `string` | 参数类型（string / number / boolean / object / array） |
| `parameters[].description` | 是 | `string` | 参数说明 |
| `parameters[].required` | 是 | `boolean` | 是否必需 |
| `max_iterations` | 否 | `int` | SubAgent 执行时的最大 ReAct 迭代次数，默认 5 |
| `llm_config` | 否 | `object` | 可选独立 LLM 配置，不配置则继承全局 |

#### 2.2.3 正文内容说明

Frontmatter 之后的 Markdown 正文是 Skill 的详细说明，面向两个读者：

1. **PlanAgent（LLM）**：通过 SkillRegister 提供的摘要了解 Skill 能力，在任务分解时根据"功能概述"和"适用场景"判断是否匹配
2. **SubAgent（LLM）**：在接收到任务后，将 SKILL.md 的正文作为系统提示词的一部分，指导其按照"执行步骤"完成工作

正文应包含以下部分（非强制，但推荐）：

- **功能概述**：Skill 的核心能力描述
- **适用场景**：什么情况下应该使用此 Skill
- **执行步骤**：分场景描述具体的执行流程，每个步骤中如需调用工具，写明工具名称和参数
- **限制条件**：该 Skill 的边界和约束

### 2.3 reference/ 目录

`reference/` 目录存放供 LLM 参考的补充文档：

- 当 LLM 阅读完 SKILL.md 后仍存在疑问时，可以读取 `reference/` 中的 `.md` 文件获取更多上下文
- 典型内容包括：领域知识说明、API 文档摘要、常见问题处理指南、编码规范等
- `reference/` 目录中的文件不参与 Skill 注册，仅在 SubAgent 执行时按需加载

**使用方式**：SubAgent 的系统提示词中会告知 LLM "如果需要更多信息，可以使用 `read_file` 工具读取 `skills/{skill_name}/reference/` 目录下的文档"。

### 2.4 scripts/ 目录

`scripts/` 目录存放 Skill 执行时可能需要的辅助脚本：

- 支持 Python 脚本（`.py`）和 Shell 脚本（`.sh` / `.bat`）
- 脚本不随 Skill 注册自动加载到内存，而是在 SubAgent 执行时按需由工具调用
- 脚本的执行环境与 `run_command` 工具共享同一安全白名单机制

**典型用途**：

| 脚本类型 | 示例 | 用途 |
|---------|------|------|
| 数据处理 | `describe_csv.py` | 读取 CSV 文件并输出列统计信息 |
| 格式转换 | `convert_encoding.py` | 批量转换文件编码 |
| 环境检查 | `check_deps.sh` | 检查必要的系统依赖是否安装 |
| 模板生成 | `gen_report.py` | 根据数据生成 Markdown 报告模板 |

```python
# src/graph_agent/skill/models.py

from dataclasses import dataclass, field
from pathlib import Path


@dataclass
class ToolParamDef:
    """工具参数定义"""
    name: str
    type: str                    # string / number / boolean / object / array
    description: str
    required: bool = True


@dataclass
class SkillToolDef:
    """Skill 中声明的工具定义"""
    name: str                    # 对应 ToolCenter 中的工具名称
    description: str             # 功能描述（用于生成 SubAgent 提示词）
    parameters: list[ToolParamDef] = field(default_factory=list)


@dataclass
class SkillMeta:
    """SKILL.md frontmatter 解析结果"""
    name: str                    # Skill 唯一标识
    description: str             # 功能描述
    tools: list[SkillToolDef] = field(default_factory=list)
    max_iterations: int = 5
    llm_config: dict | None = None


@dataclass
class SkillDef:
    """完整的 Skill 定义"""
    meta: SkillMeta                          # frontmatter 元数据
    body: str                                # SKILL.md 正文（Markdown，供 SubAgent 使用）
    summary: str                             # 摘要（供 PlanAgent 使用，从 body 中提取）
    folder_path: Path                        # Skill 文件夹绝对路径
    reference_dir: Path | None               # reference/ 目录路径（可为 None）
    scripts_dir: Path | None                 # scripts/ 目录路径（可为 None）
    reference_files: list[Path] = field(default_factory=list)    # reference/ 下的文件列表
    script_files: list[Path] = field(default_factory=list)       # scripts/ 下的文件列表
```

### 3.3 SKILL.md 解析器 (`parser.py`)

`SkillParser` 负责解析 SKILL.md 文件，分离 frontmatter 和正文，并提取 PlanAgent 所需的摘要信息。

```python
# src/graph_agent/skill/parser.py

import re
import yaml
from pathlib import Path
from graph_agent.skill.models import SkillMeta, SkillToolDef, ToolParamDef


class SkillParser:
    """SKILL.md 文件解析器。

    解析 SKILL.md 的 frontmatter（YAML）和正文（Markdown），
    返回结构化的 SkillMeta 和原始正文。
    """

    # frontmatter 分隔符
    _FRONTMATTER_RE = re.compile(r'^---\s*\n(.*?)\n---\s*\n', re.DOTALL)

    def parse(self, file_path: Path) -> tuple[SkillMeta, str, str]:
        """解析 SKILL.md 文件。

        Args:
            file_path: SKILL.md 文件的绝对路径。

        Returns:
            (meta, body, summary) 三元组:
            - meta: 解析后的 frontmatter 元数据
            - body: 完整的 Markdown 正文（去除 frontmatter）
            - summary: 供 PlanAgent 使用的摘要文本

        Raises:
            FileNotFoundError: 文件不存在
            ValueError: frontmatter 解析失败
        """
        content = file_path.read_text(encoding="utf-8")

        m = self._FRONTMATTER_RE.match(content)
        if not m:
            raise ValueError(f"{file_path}: 缺少有效的 YAML frontmatter (--- ... ---)")

        raw_yaml = m.group(1)
        body = content[m.end():].strip()

        meta = self._parse_frontmatter(raw_yaml)
        summary = self._extract_summary(body)

        return meta, body, summary

    def _parse_frontmatter(self, raw_yaml: str) -> SkillMeta:
        """解析 frontmatter YAML 为 SkillMeta。"""
        data = yaml.safe_load(raw_yaml)

        if not data:
            raise ValueError("frontmatter 为空")
        if "name" not in data:
            raise ValueError("frontmatter 缺少必填字段 'name'")
        if "description" not in data:
            raise ValueError("frontmatter 缺少必填字段 'description'")

        tools = []
        for t in data.get("tools", []):
            params = [
                ToolParamDef(
                    name=p["name"],
                    type=p.get("type", "string"),
                    description=p.get("description", ""),
                    required=p.get("required", True),
                )
                for p in t.get("parameters", [])
            ]
            tools.append(SkillToolDef(
                name=t["name"],
                description=t.get("description", ""),
                parameters=params,
            ))

        return SkillMeta(
            name=data["name"],
            description=data["description"],
            tools=tools,
            max_iterations=data.get("max_iterations", 5),
            llm_config=data.get("llm_config"),
        )

    def _extract_summary(self, body: str) -> str:
        """从 body 中提取供 PlanAgent 使用的功能摘要。

        提取策略：
        1. 找到 "## 功能概述" 标题后的第一个段落
        2. 如不存在，取 "## 适用场景" 后的内容
        3. 如都不存在，取 body 的前 200 个字符
        """
        # 尝试提取 "功能概述" 段落
        overview_match = re.search(
            r'##\s*功能概述\s*\n+(.+?)(?=\n##|\Z)',
            body, re.DOTALL
        )
        if overview_match:
            return overview_match.group(1).strip()

        # 尝试提取 "适用场景" 段落
        scenario_match = re.search(
            r'##\s*适用场景\s*\n+(.+?)(?=\n##|\Z)',
            body, re.DOTALL
        )
        if scenario_match:
            return scenario_match.group(1).strip()

        # 回退：取正文前 200 字符
        return body[:200].strip()
```

### 3.4 SkillLoader (`loader.py`)

`SkillLoader` 负责在 GraphAgent 启动时扫描 `skills/` 目录，发现所有包含 `SKILL.md` 的子目录，解析并加载为 `SkillDef` 对象。

```python
# src/graph_agent/skill/loader.py

import os
from pathlib import Path
from graph_agent.skill.parser import SkillParser
from graph_agent.skill.models import SkillDef


class SkillLoader:
    """Skill 加载器——扫描 skills/ 目录并加载所有 Skill。

    使用方式:
        loader = SkillLoader()
        skills = loader.load_all()
        # skills: list[SkillDef]
    """

    def __init__(self, skills_root: str | None = None):
        """
        Args:
            skills_root: skills/ 目录的绝对路径。
                        默认为项目根目录下的 skills/。
                        可通过环境变量 SKILLS_ROOT 覆盖。
        """
        if skills_root is None:
            skills_root = os.getenv("SKILLS_ROOT")
        if skills_root is None:
            project_root = Path(__file__).parent.parent.parent.parent
            skills_root = str(project_root / "skills")
        self._root = Path(skills_root)
        self._parser = SkillParser()

    def load_all(self) -> list[SkillDef]:
        """扫描并加载所有 Skill。

        扫描逻辑：
        1. 遍历 skills/ 下的所有子目录
        2. 查找每个子目录中的 SKILL.md 文件
        3. 解析 SKILL.md，构建 SkillDef
        4. 扫描 reference/ 和 scripts/ 子目录（如存在）
        5. 跳过解析失败的 Skill 并记录警告日志

        Returns:
            成功加载的 SkillDef 列表
        """
        skills: list[SkillDef] = []

        if not self._root.exists():
            return skills

        for skill_dir in sorted(self._root.iterdir()):
            if not skill_dir.is_dir():
                continue

            skill_md = skill_dir / "SKILL.md"
            if not skill_md.exists():
                continue

            try:
                skill_def = self._load_one(skill_dir, skill_md)
                skills.append(skill_def)
            except Exception as e:
                # 记录警告但不中断整个加载过程
                import logging
                logging.getLogger("graph_agent").warning(
                    f"加载 Skill 失败: {skill_dir.name} — {e}"
                )

        return skills

    def _load_one(self, skill_dir: Path, skill_md: Path) -> SkillDef:
        """加载单个 Skill 文件夹。"""
        meta, body, summary = self._parser.parse(skill_md)

        # 扫描 reference/ 目录
        reference_dir = skill_dir / "reference"
        reference_files = []
        if reference_dir.exists() and reference_dir.is_dir():
            reference_files = sorted(
                p for p in reference_dir.iterdir()
                if p.is_file() and p.suffix in (".md", ".txt")
            )

        # 扫描 scripts/ 目录
        scripts_dir = skill_dir / "scripts"
        script_files = []
        if scripts_dir.exists() and scripts_dir.is_dir():
            script_files = sorted(
                p for p in scripts_dir.iterdir()
                if p.is_file() and p.suffix in (".py", ".sh", ".bat")
            )

        return SkillDef(
            meta=meta,
            body=body,
            summary=summary,
            folder_path=skill_dir,
            reference_dir=reference_dir if reference_files else None,
            scripts_dir=scripts_dir if script_files else None,
            reference_files=reference_files,
            script_files=script_files,
        )

    def reload(self) -> list[SkillDef]:
        """重新扫描并加载所有 Skill（热加载）。"""
        return self.load_all()

    @property
    def root(self) -> Path:
        return self._root
```

### 3.5 SkillRegister (`register.py`)

`SkillRegister` 维护所有已加载的 Skill，提供按名称查找、按功能描述匹配、生成 PlanAgent 摘要等能力。同时负责将 Skill 转换为 SubAgent 配置，与 `SubAgentRegistry` 集成。

```python
# src/graph_agent/skill/register.py

from graph_agent.skill.models import SkillDef
from graph_agent.skill.loader import SkillLoader


class SkillRegister:
    """Skill 注册中心——维护所有可用的 Skill。

    职责：
    1. 持有所有已加载的 SkillDef
    2. 提供按名称查找、按描述匹配的能力
    3. 生成供 PlanAgent 使用的 Skill 清单
    4. 将 Skill 转换为 SubAgentConfig 供 SubAgentRegistry 使用
    """

    def __init__(self, loader: SkillLoader | None = None):
        self._skills: dict[str, SkillDef] = {}
        self._loader = loader or SkillLoader()
        self._load()

    def _load(self) -> None:
        """从 SkillLoader 加载所有 Skill 并注册。"""
        self._skills.clear()
        for skill in self._loader.load_all():
            self._skills[skill.meta.name] = skill

    def reload(self) -> None:
        """热加载：重新扫描 skills/ 目录。"""
        self._loader.reload()
        self._load()

    def get(self, name: str) -> SkillDef:
        """按名称获取 Skill。"""
        if name not in self._skills:
            raise KeyError(f"Skill 未注册: {name}")
        return self._skills[name]

    def find_by_description(self, query: str) -> list[SkillDef]:
        """根据功能描述模糊匹配 Skill。

        匹配策略：
        1. 精确匹配：query 完全等于 Skill 名称
        2. 关键词匹配：query 中的关键词出现在 Skill 的 description 或 summary 中
        3. 返回匹配的 Skill 列表，按相关度排序

        注意：此方法用于 PlanAgent 在任务分解时通过 LLM 推理选中的 Skill 名称
        来精确查找。模糊搜索能力保留给未来扩展（如 embedding 语义匹配）。
        """
        query_lower = query.lower()

        # 精确匹配名称
        for skill in self._skills.values():
            if skill.meta.name.lower() == query_lower:
                return [skill]

        # 关键词匹配
        results = []
        for skill in self._skills.values():
            desc_lower = skill.meta.description.lower()
            summary_lower = skill.summary.lower()
            if query_lower in desc_lower or query_lower in summary_lower:
                results.append(skill)

        return results

    def list_all(self) -> list[SkillDef]:
        """列出所有已注册的 Skill。"""
        return list(self._skills.values())

    def describe_all_for_llm(self) -> str:
        """生成供 PlanAgent 使用的 Skill 清单。

        与 SubAgentRegistry.describe_all_for_llm() 保持相同格式，
        确保 PlanAgent 可以无差别处理手动配置的 SubAgent 和 Skill 自动生成的 SubAgent。
        """
        if not self._skills:
            return "（暂无可用 Skill）"

        lines = []
        for skill in self._skills.values():
            # 格式与 SubAgentRegistry.describe_all_for_llm() 保持一致:
            # - **agent_name**: description（技能: skill_name）
            lines.append(
                f"- **{skill.meta.name}**: {skill.meta.description}"
            )
        return "\n".join(lines)

    def to_subagent_configs(self) -> list[dict]:
        """将已注册的 Skill 转换为 SubAgent 配置字典列表。

        每个 Skill 生成一条 SubAgent 配置，遵循 config/subagents.yaml 的格式。
        这使得 Skill 可以无缝集成到现有的 SubAgentRegistry 中。
        """
        configs = []
        for skill in self._skills.values():
            configs.append({
                "name": skill.meta.name,
                "description": skill.meta.description,
                "skills": [skill.meta.name],
                "tools": [t.name for t in skill.meta.tools],
                "prompt_file": skill.meta.name,   # 动态生成提示词，非文件加载
                "max_iterations": skill.meta.max_iterations,
                "llm_config": skill.meta.llm_config,
                "_source": "skill",               # 标记来源：skill 自动生成
                "_skill_def": skill,              # 持有完整的 SkillDef 引用
            })
        return configs
```

### 4.1 SubAgentRegistry 集成

```python
# src/graph_agent/orchestration/subagent.py 中的新增/修改逻辑

class SubAgentRegistry:
    def __init__(self, config_path=None, prompt_loader=None, skill_register=None):
        self._agents: dict[str, SubAgentConfig] = {}
        self._prompt_loader = prompt_loader or PromptLoader()
        self._config_path = config_path or self._default_config_path()
        self._skill_register = skill_register  # 新增：SkillRegister 引用
        self._load_all()

    def _load_all(self) -> None:
        """加载所有 SubAgent：先加载 YAML 配置，再加载 Skill 定义。
        Skill 定义的同名 SubAgent 会覆盖 YAML 中的定义。
        """
        self._load_from_config()        # 现有逻辑
        if self._skill_register:
            self._load_from_skills()    # 新增逻辑

    def _load_from_skills(self) -> None:
        """从 SkillRegister 加载 Skill 并转换为 SubAgent 配置。"""
        for skill_config in self._skill_register.to_subagent_configs():
            config = SubAgentConfig(
                name=skill_config["name"],
                description=skill_config["description"],
                skills=skill_config["skills"],
                tools=skill_config["tools"],
                prompt_file=skill_config["prompt_file"],
                max_iterations=skill_config.get("max_iterations", 5),
                llm_config=skill_config.get("llm_config"),
            )
            # 存储额外引用，供 SubAgent 执行时获取完整 SKILL.md 正文
            config._skill_def = skill_config.get("_skill_def")
            self._agents[config.name] = config  # 同名覆盖 YAML 定义
```

### 4.2 SubAgent 提示词生成

Skill 自动生成的 SubAgent 不使用 `prompts/` 固定文件，而是在运行时动态构建系统提示词：

```python
# SubAgentConfig 新增方法（或独立函数）

def build_skill_system_prompt(skill_def: SkillDef, task_description: str) -> str:
    """根据 SkillDef 构建 SubAgent 的系统提示词。"""
    meta = skill_def.meta

    tools_desc = ""
    if meta.tools:
        tool_lines = []
        for t in meta.tools:
            params_desc = ", ".join(
                f"{p.name}: {p.type}" + ("?" if not p.required else "")
                for p in t.parameters
            )
            tool_lines.append(f"- **{t.name}**({params_desc}): {t.description}")
        tools_desc = "## 可用工具\n" + "\n".join(tool_lines)

    ref_desc = ""
    if skill_def.reference_files:
        ref_paths = "\n".join(
            f"  - skills/{skill_def.meta.name}/reference/{f.name}"
            for f in skill_def.reference_files
        )
        ref_desc = (
            f"\n## 参考文档\n"
            f"如果需要更多信息，可以使用 `read_file` 工具读取以下参考文档：\n{ref_paths}"
        )

    return f"""你是一个 {meta.name} 专家。

## 能力说明
{meta.description}

## 执行指南
{skill_def.body}

{tools_desc}{ref_desc}

## 当前任务
{task_description}

请按照执行指南中的步骤完成任务。如果遇到指南未覆盖的情况，根据你的专业知识灵活处理，但不要超出能力说明中定义的范围。
"""
```

## 6. 关键设计决策

### 6.1 为什么选择 Markdown 而非纯 YAML？

SKILL.md 采用 "YAML frontmatter + Markdown 正文" 的混合格式：

- **Frontmatter**：承载机器可解析的结构化元数据（名称、工具声明、参数配置），便于 `SkillParser` 自动提取
- **Markdown 正文**：承载面向 LLM 的自然语言指令（功能说明、执行步骤），LLM 原生理解 Markdown 格式，无需额外解析

纯 YAML 的问题在于多段落、多列表的"执行步骤"难以阅读和维护；纯 Markdown 的问题在于缺少结构化元数据，解析器难以提取工具声明。

### 6.2 为什么 Skill 与 SubAgent 是 1:1 映射？

当前设计中，每个 Skill 对应一个 SubAgent。理由：

1. **关注点分离**：每个 Skill 有独立的执行步骤、工具集和限制条件，对应一个专门的 SubAgent 执行者，职责清晰
2. **提示词隔离**：每个 SubAgent 的系统提示词由对应 SKILL.md 生成，不同 Skill 的指令不会互相干扰
3. **匹配简化**：PlanAgent 通过 Skill 名称即可精确匹配 SubAgent，无需处理"一个 Agent 具备多种组合技能"的歧义

未来可扩展为 N:1 映射（一个 SubAgent 具备多个 Skill），但这需要更复杂的提示词编排和任务路由逻辑，当前阶段保持简单。

### 6.3 为什么 `max_iterations` 放在 Skill 层面而非全局？

不同 Skill 的复杂度差异很大：

- `time_query`：获取当前时间，通常 1 次迭代即可完成
- `data-analysis`：可能需要读取文件 → 执行脚本 → 计算 → 写报告，需要多轮迭代

将 `max_iterations` 放在 Skill 定义中，允许每个 Skill 根据自己的复杂度设置合理的迭代上限，避免简单任务浪费 token 或复杂任务过早终止。

### 6.4 脚本安全模型

`scripts/` 目录中的脚本执行遵循与 `run_command` 工具相同的安全策略（命令白名单 + 沙箱限制）。此外：

- 脚本路径限定在 `skills/{skill_name}/scripts/` 范围内，不允许访问其他 Skill 的脚本
- SubAgent 系统提示词中明确告知脚本目录路径，由 LLM 自行判断是否需要调用
- 脚本不随 Skill 注册自动加载到内存，避免引入依赖污染
