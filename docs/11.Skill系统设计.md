# GraphAgent Skill 系统设计文档

## 1. 概述

### 1.1 设计背景

当前 GraphAgent 的 SubAgent 系统通过 `config/subagents.yaml` 手动配置，存在以下局限：

- **配置分散**：SubAgent 的元数据（名称、描述、技能标签）在 YAML 中，提示词在 `prompts/` 目录下，两者分离管理，新增一个 SubAgent 需要在多处操作
- **能力描述不够丰富**：YAML 中的 `skills` 仅为简单的字符串标签，PlanAgent 只能看到一个技能名称，无法理解该技能的详细执行步骤、适用场景和限制条件
- **缺乏自包含性**：一个完整的"技能"（如 PDF 解析）除了提示词外，往往还需要示例文档（reference）和辅助脚本（scripts），现有结构无法将这三者组织在一起
- **扩展门槛高**：对于复杂技能，用户需要同时理解 YAML 配置格式、提示词文件路径约定、工具注册机制等多个概念

Skill 系统将技能定义从"配置驱动"升级为"文档驱动"——每个 Skill 是一个自包含的文件夹，内含功能说明、执行步骤、参考示例和辅助脚本。系统在启动时自动扫描并注册所有 Skill，根据 SKILL.md 中的描述自动生成对应的 SubAgent。

### 1.2 核心设计理念

```
现有方式（配置驱动）:
  config/subagents.yaml  →  手写 YAML 配置  →  SubAgent
  prompts/subagent/*.txt →  手写提示词     →  系统提示词

Skill 方式（文档驱动）:
  skills/{skill_name}/SKILL.md      →  技能定义文档  →  自动生成 SubAgent 配置 + 系统提示词
  skills/{skill_name}/reference/   →  参考示例     →  辅助 LLM 理解
  skills/{skill_name}/scripts/     →  辅助脚本     →  执行时调用
```

### 1.3 与现有系统的关系

Skill 系统**完全替代**现有基于 `config/subagents.yaml` 的 SubAgent 系统：

- `config/subagents.yaml` 及 `prompts/subagent/` 目录不再使用，所有 SubAgent 均来源于 Skill
- `SubAgentRegistry` 不再从 YAML 配置文件加载，改为从 `SkillRegister` 获取全部 SubAgent 配置
- PlanAgent 的任务分解和派发逻辑保持不变，仍通过 `SubAgentRegistry.find_by_skill()` 匹配 SubAgent——只是 SubAgent 的来源从 YAML 变为 Skill
- `prompts/guard/` 和 `prompts/plan/` 目录保留（GuardAgent 和 PlanAgent 的提示词与 Skill 无关）

### 1.4 Skill 分类

GraphAgent 中的 Skill 分为两种类型，按来源和复杂程度区分：

| 类型 | 来源 | 目录位置 | 文件组成 | 工具来源 |
|------|------|---------|---------|---------|
| **内置 Skill** | GraphAgent 自带 | `src/graph_agent/skills/` | 一个 `.md` 文件（文件名即 Skill 名） | `graph_agent.tools` 模块中已注册的工具 |
| **用户自定义 Skill** | 用户创建 | 项目根目录 `skills/` | `SKILL.md` + 可选 `reference/` + 可选 `scripts/` | `graph_agent.tools` 模块中的工具 + `scripts/` 中的定制脚本 |

**内置 Skill** 特点：
- 随 GraphAgent 包分发，提供基础能力（如文件操作、计算、时间查询、网页抓取等）
- 结构精简，一个 `.md` 文件即为一个 Skill，文件名即 Skill 名称（如 `file-management.md`），无需子文件夹、无 `reference/` 和 `scripts/`
- `.md` 文件中声明的工具均在 `graph_agent.tools` 模块中有对应实现
- 用户无需关心内置 Skill 的底层工具实现

**用户自定义 Skill** 特点：
- 由用户按需创建，放置在项目根目录的 `skills/` 下
- 可引用 `graph_agent.tools` 模块中的现有工具
- 如果需要 GraphAgent 工具模块中不存在的工具或脚本，用户需在 Skill 文件夹下创建 `scripts/` 子目录存放定制脚本（`.py` / `.sh`）
- Skill 加载时，`scripts/` 中的脚本会被自动注册为可供该 SubAgent 调用的工具
- 如果脚本实现的功能需要被多个 Skill 复用，应将其贡献到 `graph_agent.tools` 模块中，而非在各 Skill 的 `scripts/` 中重复维护

```
内置 Skill (自带)                      用户自定义 Skill
─────────────────────────          ─────────────────────────
src/graph_agent/skills/            skills/
├── file-management.md             ├── data-analysis/
├── calculation.md                 │   ├── SKILL.md
├── time-query.md                  │   ├── reference/
├── command-execution.md           │   │   └── stats-guide.md
├── web-fetch.md                   │   └── scripts/
├── json-processing.md             │       └── describe_csv.py
└── text-generation.md             ├── code-review/
                                   │   ├── SKILL.md
                                   │   └── reference/
                                   │       └── security-checklist.md
                                   └── custom-workflow/
                                       ├── SKILL.md
                                       └── scripts/
                                           ├── step1_preprocess.py
                                           └── step3_upload.sh
```

## 2. Skill 目录结构

### 2.1 总体布局

Skill 分布在两个目录中，SkillLoader 在启动时依次扫描：

```
# 内置 Skill（随 GraphAgent 包分发）—— 一个 .md 文件即为一个 Skill
src/graph_agent/skills/              # 内置 Skill 根目录

├── calculation.md
├── time-query.md

# 用户自定义 Skill（用户按需创建）—— 一个子文件夹对应一个 Skill
skills/                              # 项目根目录下的用户 Skill 根目录（可配置）
├── data-analysis/                   # 含定制脚本的用户 Skill
│   ├── SKILL.md                     # [必需] Skill 定义文件
│   ├── reference/                   # [可选] 参考示例目录
│   │   └── stats-guide.md           # 统计分析指南

    ├── SKILL.md

        └── step3_upload.sh          # 上传脚本
```

### 2.2 SKILL.md 文件规范

SKILL.md 是 Skill 的核心定义文件，采用 Markdown 格式，包含标准的 frontmatter（YAML 头部）和正文内容。

#### 2.2.1 完整格式（用户自定义 Skill 示例）

```markdown
---
name: data-analysis
type: user
description: 对结构化数据进行统计分析，支持 CSV/JSON 数据读取、数据清洗、统计计算和报告生成
tools:
  - name: read_file
    description: 读取数据文件内容
    parameters:
      - name: path
        type: string
        description: 数据文件路径
        required: true
  - name: safe_calculator
    description: 执行安全的数学表达式计算
    parameters:
      - name: expression
        type: string
        description: 数学表达式字符串
        required: true
  - name: run_command
    description: 执行 scripts/ 目录下的定制脚本
    parameters:
      - name: command
        type: string
        description: 要执行的命令
        required: true
  - name: write_file
    description: 将分析结果写入文件
    parameters:
      - name: path
        type: string
        description: 输出文件路径
        required: true
      - name: content
        type: string
        description: 要写入的内容
        required: true
max_iterations: 5

# 数据分析技能

## 功能概述
对 CSV、JSON 等结构化数据文件进行统计分析，包括数据读取、探索性分析、统计计算和结果报告生成。

## 适用场景
- 用户提供了 CSV/JSON 数据文件，需要进行统计分析
- 需要计算数据的均值、中位数、标准差等统计指标
- 需要从数据中提取特定模式或趋势
- 需要生成数据分析报告

## 执行步骤

### 场景一：CSV 数据探索分析
1. 使用 `read_file` 读取 CSV 文件，了解数据结构
2. 使用 `run_command` 执行 `python skills/data-analysis/scripts/describe_csv.py <文件路径>` 获取列统计信息
3. 根据统计信息识别数据特征（缺失值、异常值、分布特征）
4. 使用 `safe_calculator` 进行必要的数值计算
5. 汇总分析结果，生成结构化报告

### 场景二：JSON 数据处理
1. 使用 `read_file` 读取 JSON 文件
2. 解析 JSON 结构，识别嵌套层级和数据类型
3. 根据用户需求提取或聚合数据
4. 使用 `write_file` 保存处理结果
5. 返回处理摘要

## 限制条件
- 数据文件超过 100MB 时提示用户确认
- 不直接修改原始数据文件（只读分析）
- 定制脚本仅存放在 `skills/data-analysis/scripts/` 目录下
```

#### 2.2.2 Frontmatter 字段说明

| 字段 | 必需 | 类型 | 说明 |
|------|------|------|------|
| `name` | 是 | `string` | Skill 唯一标识名称，对应 SubAgent 名称（如 `data-analysis`） |
| `type` | 是 | `string` | Skill 类型：`builtin`（内置 Skill，仅 SKILL.md）或 `user`（用户自定义 Skill） |
| `description` | 是 | `string` | Skill 功能的一句话描述，供 PlanAgent 理解和匹配 |
| `tools` | 否 | `list[ToolDef]` | 该 Skill 需要使用的工具列表。内置 Skill 的工具必须在 `graph_agent.tools` 模块中存在；用户自定义 Skill 可引用模块中的工具，也可使用 `scripts/` 中的定制脚本 |
| `tools[].name` | 是 | `string` | 工具名称。对于内置 Skill 和模块工具，对应 `ToolCenter` 中注册的名称；对于用户自定义 Skill 的定制脚本，对应脚本文件名（如 `describe_csv.py`） |
| `tools[].description` | 是 | `string` | 工具的功能描述，用于生成 SubAgent 系统提示词 |
| `tools[].parameters` | 是 | `list[ParamDef]` | 工具的参数列表 |
| `parameters[].name` | 是 | `string` | 参数名称 |
| `parameters[].type` | 是 | `string` | 参数类型（string / number / boolean / object / array） |
| `parameters[].description` | 是 | `string` | 参数说明 |
| `parameters[].required` | 是 | `boolean` | 是否必需 |
| `max_iterations` | 否 | `int` | SubAgent 执行时的最大 ReAct 迭代次数，默认 5 |
| `llm_config` | 否 | `object` | 可选独立 LLM 配置，不配置则继承全局 |

#### 2.2.3 正文内容说明

Frontmatter 之后的 Markdown 正文是 Skill 的详细说明，面向两个读者：

1. **PlanAgent（LLM）**：通过 SkillRegister 提供的摘要了解 Skill 能力，在任务分解时根据"功能概述"和"适用场景"判断是否匹配
2. **SubAgent（LLM）**：在接收到任务后，将 SKILL.md 的正文作为系统提示词的一部分，指导其按照"执行步骤"完成工作

正文应包含以下部分（非强制，但推荐）：

- **功能概述**：Skill 的核心能力描述
- **适用场景**：什么情况下应该使用此 Skill
- **执行步骤**：分场景描述具体的执行流程，每个步骤中如需调用工具，写明工具名称和参数
- **限制条件**：该 Skill 的边界和约束

### 2.3 内置 Skill 格式

内置 Skill 结构精简，一个 `.md` 文件即为一个 Skill，文件名即 Skill 名称，直接放在 `src/graph_agent/skills/` 目录下，无需子文件夹。其声明的工具全部来自 `graph_agent.tools` 模块。

```markdown
        required: true

        type: string
        description: 要写入的内容
        required: true
  - name: list_directory
    description: 列出目录内容

      - name: path
        type: string

# 文件管理技能

## 功能概述
对本地文件系统进行安全的读写操作，支持文本文件的常见操作和目录管理。

## 适用场景
- 需要读取文件内容并提取信息
- 需要将生成的结果保存到文件
- 需要列出目录结构或检查文件是否存在

## 执行步骤

### 场景一：读取并分析文件内容
1. 使用 `read_file` 工具读取目标文件内容
2. 分析文件内容，提取用户需要的信息
3. 如果文件编码异常，尝试不同编码重新读取
4. 将分析结果返回

### 场景二：写入文件
1. 确认目标目录是否存在，必要时使用 `list_directory` 检查
2. 使用 `write_file` 工具将内容写入指定路径
3. 使用 `read_file` 工具验证写入结果
4. 返回写入结果确认

## 限制条件
- 不会删除包含重要系统文件的目录
- 写入前会确认目标路径的安全性和合法性
- 大文件（>10MB）读取时会提示用户确认
```

### 2.4 reference/ 目录（仅用户自定义 Skill）

`reference/` 目录存放供 LLM 参考的补充文档，仅用户自定义 Skill 可包含此目录：

- 当 LLM 阅读完 SKILL.md 后仍存在疑问时，可以读取 `reference/` 中的 `.md` 文件获取更多上下文
- 典型内容包括：领域知识说明、API 文档摘要、常见问题处理指南、编码规范等
- `reference/` 目录中的文件不参与 Skill 注册，仅在 SubAgent 执行时按需加载
- 内置 Skill 不包含 `reference/` 目录——其功能说明在 SKILL.md 正文中已完备

**使用方式**：SubAgent 的系统提示词中会告知 LLM "如果需要更多信息，可以使用 `read_file` 工具读取 `skills/{skill_name}/reference/` 目录下的文档"。

### 2.5 scripts/ 目录（仅用户自定义 Skill）

`scripts/` 目录存放 GraphAgent `tools` 模块中不存在的定制工具或脚本，仅用户自定义 Skill 可包含此目录：

- **设计意图**：当用户自定义 Skill 需要的能力在 `graph_agent.tools` 模块中找不到对应工具时，通过 `scripts/` 提供定制实现
- 支持 Python 脚本（`.py`）和 Shell 脚本（`.sh` / `.bat`）
- Skill 加载时，`scripts/` 中的脚本文件会被自动注册为可供该 Skill 对应 SubAgent 调用的工具
- 脚本的执行环境与 `run_command` 工具共享同一安全白名单机制
- 内置 Skill 不包含 `scripts/` 目录——其使用的全部工具均来自 `graph_agent.tools` 模块

**典型用途**：

| 脚本类型 | 示例 | 对应场景 |
|---------|------|---------|
| 数据处理 | `describe_csv.py` | `tools` 模块中没有 CSV 统计工具，用户自行编写 |
| 格式转换 | `convert_encoding.py` | 批量文件编码转换，`tools` 模块未覆盖 |
| 环境检查 | `check_deps.sh` | 检查特定系统依赖是否安装 |
| API 调用 | `fetch_api.py` | 调用特定第三方 API，`tools` 模块的 `fetch_web` 不满足需求 |

**注意**：如果某个脚本实现的功能具有通用性（可被多个 Skill 使用），应将其贡献到 `graph_agent.tools` 模块中成为正式工具，而非在各 Skill 的 `scripts/` 中重复维护。

### 3.2 数据模型 (`models.py`)

```python
# src/graph_agent/skill/models.py

from dataclasses import dataclass, field
from pathlib import Path


@dataclass
class ToolParamDef:
    """工具参数定义"""
    name: str
    type: str                    # string / number / boolean / object / array
    description: str
    required: bool = True


@dataclass
class SkillToolDef:
    """Skill 中声明的工具定义"""
    name: str                    # 对应 ToolCenter 中的工具名称
    description: str             # 功能描述（用于生成 SubAgent 提示词）
    parameters: list[ToolParamDef] = field(default_factory=list)


@dataclass
class SkillMeta:
    """SKILL.md frontmatter 解析结果"""
    name: str                    # Skill 唯一标识
    type: str                    # Skill 类型："builtin"（内置）或 "user"（用户自定义）
    description: str             # 功能描述
    tools: list[SkillToolDef] = field(default_factory=list)
    max_iterations: int = 5
    llm_config: dict | None = None


@dataclass
class SkillDef:
    """完整的 Skill 定义"""
    meta: SkillMeta                          # frontmatter 元数据
    body: str                                # SKILL.md 正文（Markdown，供 SubAgent 使用）
    summary: str                             # 摘要（供 PlanAgent 使用，从 body 中提取）
    folder_path: Path                        # Skill 文件/文件夹路径（内置 Skill 指向 .md 文件，用户 Skill 指向子文件夹）
    reference_dir: Path | None               # reference/ 目录路径（仅 user 类型可为非 None）
    scripts_dir: Path | None                 # scripts/ 目录路径（仅 user 类型可为非 None）
    reference_files: list[Path] = field(default_factory=list)    # reference/ 下的文件列表
    script_files: list[Path] = field(default_factory=list)       # scripts/ 下的文件列表
```

### 3.3 SKILL.md 解析器 (`parser.py`)

`SkillParser` 负责解析 SKILL.md 文件，分离 frontmatter 和正文，并提取 PlanAgent 所需的摘要信息。

```python
# src/graph_agent/skill/parser.py

import re
import yaml
from pathlib import Path
from graph_agent.skill.models import SkillMeta, SkillToolDef, ToolParamDef


class SkillParser:
    """SKILL.md 文件解析器。

    解析 SKILL.md 的 frontmatter（YAML）和正文（Markdown），
    返回结构化的 SkillMeta 和原始正文。
    """

    # frontmatter 分隔符
    _FRONTMATTER_RE = re.compile(r'^---\s*\n(.*?)\n---\s*\n', re.DOTALL)

    def parse(self, file_path: Path) -> tuple[SkillMeta, str, str]:
        """解析 SKILL.md 文件。

        Args:
            file_path: SKILL.md 文件的绝对路径。

        Returns:
            (meta, body, summary) 三元组:
            - meta: 解析后的 frontmatter 元数据
            - body: 完整的 Markdown 正文（去除 frontmatter）
            - summary: 供 PlanAgent 使用的摘要文本

        Raises:
            FileNotFoundError: 文件不存在
            ValueError: frontmatter 解析失败
        """
        content = file_path.read_text(encoding="utf-8")

        m = self._FRONTMATTER_RE.match(content)
        if not m:
            raise ValueError(f"{file_path}: 缺少有效的 YAML frontmatter (--- ... ---)")

        raw_yaml = m.group(1)
        body = content[m.end():].strip()

        meta = self._parse_frontmatter(raw_yaml)
        summary = self._extract_summary(body)

        return meta, body, summary

    def _parse_frontmatter(self, raw_yaml: str) -> SkillMeta:
        """解析 frontmatter YAML 为 SkillMeta。"""
        data = yaml.safe_load(raw_yaml)

        if not data:
            raise ValueError("frontmatter 为空")
        if "name" not in data:
            raise ValueError("frontmatter 缺少必填字段 'name'")
        if "type" not in data:
            raise ValueError("frontmatter 缺少必填字段 'type'（必须为 'builtin' 或 'user'）")
        if data["type"] not in ("builtin", "user"):
            raise ValueError(f"无效的 Skill 类型 '{data['type']}'，必须为 'builtin' 或 'user'")
        if "description" not in data:
            raise ValueError("frontmatter 缺少必填字段 'description'")

        tools = []
        for t in data.get("tools", []):
            params = [
                ToolParamDef(
                    name=p["name"],
                    type=p.get("type", "string"),
                    description=p.get("description", ""),
                    required=p.get("required", True),
                )
                for p in t.get("parameters", [])
            ]
            tools.append(SkillToolDef(
                name=t["name"],
                description=t.get("description", ""),
                parameters=params,
            ))

        return SkillMeta(
            name=data["name"],
            type=data["type"],
            description=data["description"],
            tools=tools,
            max_iterations=data.get("max_iterations", 5),
            llm_config=data.get("llm_config"),
        )

    def _extract_summary(self, body: str) -> str:
        """从 body 中提取供 PlanAgent 使用的功能摘要。

        提取策略：
        1. 找到 "## 功能概述" 标题后的第一个段落
        2. 如不存在，取 "## 适用场景" 后的内容
        3. 如都不存在，取 body 的前 200 个字符
        """
        # 尝试提取 "功能概述" 段落
        overview_match = re.search(
            r'##\s*功能概述\s*\n+(.+?)(?=\n##|\Z)',
            body, re.DOTALL
        )
        if overview_match:
            return overview_match.group(1).strip()

        # 尝试提取 "适用场景" 段落
        scenario_match = re.search(
            r'##\s*适用场景\s*\n+(.+?)(?=\n##|\Z)',
            body, re.DOTALL
        )
        if scenario_match:
            return scenario_match.group(1).strip()

        # 回退：取正文前 200 字符
        return body[:200].strip()
```

### 3.4 SkillLoader (`loader.py`)

`SkillLoader` 负责在 GraphAgent 启动时扫描 `skills/` 目录，发现所有包含 `SKILL.md` 的子目录，解析并加载为 `SkillDef` 对象。

```python
# src/graph_agent/skill/loader.py

import os
from pathlib import Path
from graph_agent.skill.parser import SkillParser
from graph_agent.skill.models import SkillDef


class SkillLoader:
    """Skill 加载器——扫描内置 Skill 和用户自定义 Skill 并加载。

    内置 Skill：src/graph_agent/skills/ 下的 .md 文件（一个文件 = 一个 Skill）。
    用户 Skill：项目根目录 skills/ 下的子文件夹（一个文件夹 = 一个 Skill，含 SKILL.md）。

    加载顺序：先加载内置 Skill，再加载用户 Skill。
    如遇同名 Skill，用户 Skill 覆盖内置 Skill（允许用户定制）。

    使用方式:
        loader = SkillLoader()
        skills = loader.load_all()
        # skills: list[SkillDef]
    """

    def __init__(self, user_skills_root: str | None = None):
        """
        Args:
            user_skills_root: 用户 Skill 根目录的绝对路径。
                             默认为项目根目录下的 skills/。
                             可通过环境变量 SKILLS_ROOT 覆盖。
        """
        # 内置 Skill 目录：随包分发，不可配置
        self._builtin_root = Path(__file__).parent.parent / "skills"

        if user_skills_root is None:
            user_skills_root = os.getenv("SKILLS_ROOT")
        if user_skills_root is None:
            project_root = Path(__file__).parent.parent.parent.parent
            user_skills_root = str(project_root / "skills")
        self._user_root = Path(user_skills_root)

        self._parser = SkillParser()

    def load_all(self) -> list[SkillDef]:
        """扫描并加载所有 Skill（内置 + 用户自定义）。

        加载策略：
        1. 先扫描内置 Skill：src/graph_agent/skills/*.md
        2. 再扫描用户 Skill：skills/*/SKILL.md
        3. 同名 Skill 以用户定义的为准（允许覆盖内置 Skill）
        4. 跳过解析失败的 Skill 并记录警告日志

        Returns:
            成功加载的 SkillDef 列表
        """
        skills: dict[str, SkillDef] = {}

        # 第一轮：加载内置 Skill（扫描 .md 文件）
        for skill_def in self._scan_builtin():
            skills[skill_def.meta.name] = skill_def

        # 第二轮：加载用户 Skill（扫描子文件夹，同名覆盖）
        for skill_def in self._scan_user():
            if skill_def.meta.name in skills:
                import logging
                logging.getLogger("graph_agent").info(
                    f"用户 Skill '{skill_def.meta.name}' 覆盖同名的内置 Skill"
                )
            skills[skill_def.meta.name] = skill_def

        return list(skills.values())

    def _scan_builtin(self) -> list[SkillDef]:
        """扫描内置 Skill：src/graph_agent/skills/*.md。

        每个 .md 文件即为一个 Skill，文件名（不含扩展名）作为 Skill 名称。
        内置 Skill 的 type 自动设为 "builtin"，无需在 frontmatter 中指定（也可显式声明）。
        """
        result: list[SkillDef] = []

        if not self._builtin_root.exists():
            return result

        for md_file in sorted(self._builtin_root.glob("*.md")):
            skill_name = md_file.stem  # 文件名去掉 .md 后缀 = Skill 名称
            try:
                meta, body, summary = self._parser.parse(md_file)

                # 自动修正：内置 Skill 的 type 强制为 "builtin"
                if meta.type != "builtin":
                    import logging
                    logging.getLogger("graph_agent").warning(
                        f"内置 Skill '{skill_name}' 的 type 字段为 '{meta.type}'，"
                        f"已自动修正为 'builtin'"
                    )
                    meta.type = "builtin"

                # 自动修正：name 以文件名为准
                if meta.name != skill_name:
                    import logging
                    logging.getLogger("graph_agent").warning(
                        f"内置 Skill 文件名 '{skill_name}.md' 与 frontmatter name "
                        f"'{meta.name}' 不一致，已以文件名为准"
                    )
                    meta.name = skill_name

                result.append(SkillDef(
                    meta=meta,
                    body=body,
                    summary=summary,
                    folder_path=md_file,  # 直接指向 .md 文件
                    reference_dir=None,
                    scripts_dir=None,
                    reference_files=[],
                    script_files=[],
                ))
            except Exception as e:
                import logging
                logging.getLogger("graph_agent").warning(
                    f"加载内置 Skill 失败: {skill_name} — {e}"
                )

        return result

    def _scan_user(self) -> list[SkillDef]:
        """扫描用户自定义 Skill：skills/*/SKILL.md。

        每个子文件夹对应一个 Skill，其中 SKILL.md 为必需文件。
        """
        result: list[SkillDef] = []

        if not self._user_root.exists():
            return result

        for skill_dir in sorted(self._user_root.iterdir()):
            if not skill_dir.is_dir():
                continue

            skill_md = skill_dir / "SKILL.md"
            if not skill_md.exists():
                continue

            try:
                meta, body, summary = self._parser.parse(skill_md)

                # 自动修正：用户 Skill 的 type 强制为 "user"
                if meta.type != "user":
                    import logging
                    logging.getLogger("graph_agent").warning(
                        f"用户 Skill '{skill_dir.name}' 的 type 字段为 '{meta.type}'，"
                        f"已自动修正为 'user'"
                    )
                    meta.type = "user"

                # 扫描 reference/ 目录
                reference_dir = skill_dir / "reference"
                reference_files = []
                if reference_dir.exists() and reference_dir.is_dir():
                    reference_files = sorted(
                        p for p in reference_dir.iterdir()
                        if p.is_file() and p.suffix in (".md", ".txt")
                    )

                # 扫描 scripts/ 目录
                scripts_dir = skill_dir / "scripts"
                script_files = []
                if scripts_dir.exists() and scripts_dir.is_dir():
                    script_files = sorted(
                        p for p in scripts_dir.iterdir()
                        if p.is_file() and p.suffix in (".py", ".sh", ".bat")
                    )

                result.append(SkillDef(
                    meta=meta,
                    body=body,
                    summary=summary,
                    folder_path=skill_dir,
                    reference_dir=reference_dir if reference_files else None,
                    scripts_dir=scripts_dir if script_files else None,
                    reference_files=reference_files,
                    script_files=script_files,
                ))
            except Exception as e:
                import logging
                logging.getLogger("graph_agent").warning(
                    f"加载用户 Skill 失败: {skill_dir.name} — {e}"
                )

        return result

    def reload(self) -> list[SkillDef]:
        """重新扫描并加载所有 Skill（热加载）。"""
        return self.load_all()

    @property
    def builtin_root(self) -> Path:
        return self._builtin_root

    @property
    def user_root(self) -> Path:
        return self._user_root
```

### 3.5 SkillRegister (`register.py`)

`SkillRegister` 维护所有已加载的 Skill，提供按名称查找、按功能描述匹配、生成 PlanAgent 摘要等能力。同时负责将 Skill 转换为 SubAgent 配置，与 `SubAgentRegistry` 集成。

```python
# src/graph_agent/skill/register.py

from graph_agent.skill.models import SkillDef
from graph_agent.skill.loader import SkillLoader


class SkillRegister:
    """Skill 注册中心——维护所有可用的 Skill。

    职责：
    1. 持有所有已加载的 SkillDef
    2. 提供按名称查找、按描述匹配的能力
    3. 生成供 PlanAgent 使用的 Skill 清单
    4. 将 Skill 转换为 SubAgentConfig 供 SubAgentRegistry 使用
    """

    def __init__(self, loader: SkillLoader | None = None):
        self._skills: dict[str, SkillDef] = {}
        self._loader = loader or SkillLoader()
        self._load()

    def _load(self) -> None:
        """从 SkillLoader 加载所有 Skill 并注册。"""
        self._skills.clear()
        for skill in self._loader.load_all():
            self._skills[skill.meta.name] = skill

    def reload(self) -> None:
        """热加载：重新扫描 skills/ 目录。"""
        self._loader.reload()
        self._load()

    def get(self, name: str) -> SkillDef:
        """按名称获取 Skill。"""
        if name not in self._skills:
            raise KeyError(f"Skill 未注册: {name}")
        return self._skills[name]

    def find_by_description(self, query: str) -> list[SkillDef]:
        """根据功能描述模糊匹配 Skill。

        匹配策略：
        1. 精确匹配：query 完全等于 Skill 名称
        2. 关键词匹配：query 中的关键词出现在 Skill 的 description 或 summary 中
        3. 返回匹配的 Skill 列表，按相关度排序

        注意：此方法用于 PlanAgent 在任务分解时通过 LLM 推理选中的 Skill 名称
        来精确查找。模糊搜索能力保留给未来扩展（如 embedding 语义匹配）。
        """
        query_lower = query.lower()

        # 精确匹配名称
        for skill in self._skills.values():
            if skill.meta.name.lower() == query_lower:
                return [skill]

        # 关键词匹配
        results = []
        for skill in self._skills.values():
            desc_lower = skill.meta.description.lower()
            summary_lower = skill.summary.lower()
            if query_lower in desc_lower or query_lower in summary_lower:
                results.append(skill)

        return results

    def list_all(self) -> list[SkillDef]:
        """列出所有已注册的 Skill。"""
        return list(self._skills.values())

    def describe_all_for_llm(self) -> str:
        """生成供 PlanAgent 使用的 Skill 清单。

        PlanAgent 通过此文本了解所有可用 Skill 及其能力，
        在任务分解时根据描述选择匹配的 Skill。
        """
        if not self._skills:
            return "（暂无可用 Skill）"

        lines = []
        for skill in self._skills.values():
            lines.append(
                f"- **{skill.meta.name}**: {skill.meta.description}"
            )
        return "\n".join(lines)

    def to_subagent_configs(self) -> list[dict]:
        """将已注册的 Skill 转换为 SubAgent 配置字典列表。

        每个 Skill 生成一条 SubAgent 配置，Skill 系统是 SubAgent 的唯一来源。

        对于用户自定义 Skill：
        - SKILL.md frontmatter 中声明的 tools 与 ToolCenter 中的工具按名称匹配
        - scripts/ 中的脚本文件自动注册为额外工具（工具名 = 脚本文件名）
        """
        configs = []
        for skill in self._skills.values():
            tool_names = [t.name for t in skill.meta.tools]

            # 用户自定义 Skill：将 scripts/ 中的脚本注册为工具
            script_tools = []
            if skill.meta.type == "user" and skill.script_files:
                for script_path in skill.script_files:
                    script_name = script_path.name
                    script_tools.append(script_name)

            configs.append({
                "name": skill.meta.name,
                "description": skill.meta.description,
                "skills": [skill.meta.name],
                "tools": tool_names,
                "script_tools": script_tools,     # 该 Skill 专属的脚本工具
                "max_iterations": skill.meta.max_iterations,
                "llm_config": skill.meta.llm_config,
                "_source": f"skill:{skill.meta.type}",  # 标记来源类型
                "_skill_def": skill,                    # 持有完整的 SkillDef 引用
            })
        return configs
```

## 4. 与现有系统的集成

### 4.1 SubAgentRegistry 重构

原 `SubAgentRegistry` 从 `config/subagents.yaml` 加载配置，现改为从 `SkillRegister` 获取全部 SubAgent 配置。YAML 配置文件和相关加载逻辑全部移除。

```python
# src/graph_agent/orchestration/subagent.py 的修改

class SubAgentRegistry:
    """SubAgent 注册中心——全部配置来自 SkillRegister。

    Skill 系统是 SubAgent 的唯一来源，config/subagents.yaml 不再使用。
    """

    def __init__(self, skill_register: SkillRegister,
                 prompt_loader: PromptLoader | None = None):
        self._agents: dict[str, SubAgentConfig] = {}
        self._prompt_loader = prompt_loader or PromptLoader()
        self._skill_register = skill_register
        self._load_from_skills()

    def _load_from_skills(self) -> None:
        """从 SkillRegister 加载所有 Skill 并转换为 SubAgent 配置。"""
        self._agents.clear()
        for skill_config in self._skill_register.to_subagent_configs():
            # 工具列表：SKILL.md 声明的工具 + scripts/ 中的脚本工具
            all_tools = list(skill_config["tools"])
            all_tools.extend(skill_config.get("script_tools", []))

            config = SubAgentConfig(
                name=skill_config["name"],
                description=skill_config["description"],
                skills=skill_config["skills"],
                tools=all_tools,
                prompt_file=skill_config["name"],  # 动态生成提示词
                max_iterations=skill_config.get("max_iterations", 5),
                llm_config=skill_config.get("llm_config"),
            )
            # 持有完整的 SkillDef 引用，供执行时构建系统提示词
            config._skill_def = skill_config.get("_skill_def")
            self._agents[config.name] = config

    def reload(self) -> None:
        """热加载：重新从 SkillRegister 加载。"""
        self._skill_register.reload()
        self._load_from_skills()

    # get(), find_by_skill(), list_all(), describe_all_for_llm() 保持不变
```

### 4.2 SubAgent 提示词生成

Skill 对应的 SubAgent 系统提示词在运行时根据 SKILL.md 动态构建，不再使用 `prompts/subagent/` 下的固定文件。

```python
def build_skill_system_prompt(skill_def: SkillDef, task_description: str) -> str:
    """根据 SkillDef 构建 SubAgent 的系统提示词。"""
    meta = skill_def.meta

    # 模块工具描述
    tools_desc = ""
    if meta.tools:
        tool_lines = []
        for t in meta.tools:
            params_desc = ", ".join(
                f"{p.name}: {p.type}" + ("?" if not p.required else "")
                for p in t.parameters
            )
            tool_lines.append(f"- **{t.name}**({params_desc}): {t.description}")
        tools_desc = "## 可用工具\n" + "\n".join(tool_lines)

    # 用户自定义 Skill 的脚本工具描述

    if skill_def.meta.type == "user" and skill_def.script_files:
        script_lines = []
        for sf in skill_def.script_files:
            script_lines.append(
                f"- **{sf.name}**: 位于 `skills/{meta.name}/scripts/{sf.name}` 的定制脚本，"
                f"通过 `run_command` 工具调用"
            )
        scripts_desc = "\n## 定制脚本\n" + "\n".join(script_lines)

    # 参考文档描述（仅用户自定义 Skill）
    ref_desc = ""
    if skill_def.reference_files:
        ref_paths = "\n".join(
            f"  - skills/{meta.name}/reference/{f.name}"
            for f in skill_def.reference_files
        )
        ref_desc = (
            f"\n## 参考文档\n"
            f"如果需要更多信息，可以使用 `read_file` 工具读取以下参考文档：\n{ref_paths}"
        )

    return f"""你是一个 {meta.name} 专家。

## 能力说明
{meta.description}

## 执行指南
{skill_def.body}

{tools_desc}{scripts_desc}{ref_desc}

{task_description}

请按照执行指南中的步骤完成任务。如果遇到指南未覆盖的情况，根据你的专业知识灵活处理，但不要超出能力说明中定义的范围。
"""
```

### 4.3 脚本工具注册

用户自定义 Skill 的 `scripts/` 目录中的脚本在 SubAgent 初始化时动态注册为可用工具：

```python
def register_script_tools(skill_def: SkillDef, tool_center: ToolCenter) -> list[str]:
    """将 Skill 的 scripts/ 中的脚本注册为临时工具。

    每个脚本被包装为一个 AgentTool，工具名 = 脚本文件名。
    工具执行时调用 `run_command` 执行脚本。

    返回注册的工具名称列表。
    """
    registered = []
    for script_path in skill_def.script_files:
        tool_name = script_path.name

        def make_script_func(sp):
            def script_func(command: str = "") -> str:
                """执行 Skill 定制脚本。command 为传递给脚本的额外参数。"""
                import subprocess
                full_cmd = f"{sp} {command}".strip()
                result = subprocess.run(
                    full_cmd, shell=True, capture_output=True, text=True, timeout=60
                )
                return result.stdout or result.stderr
            return script_func

        tool_center.register(
            name=tool_name,
            description=f"执行 Skill '{skill_def.meta.name}' 的定制脚本 {tool_name}",
            func=make_script_func(script_path),
        )
        registered.append(tool_name)

    return registered
```

```markdown
# 文件管理技能

## 功能概述
对本地文件系统进行安全的读写操作，支持文本文件的常见操作和目录管理。

## 适用场景
- 需要读取文件内容并提取信息
- 需要将生成的结果保存到文件
- 需要列出目录结构或检查文件是否存在

## 执行步骤

### 场景一：读取并分析文件内容
1. 使用 `read_file` 工具读取目标文件内容
2. 分析文件内容，提取用户需要的信息
3. 如果文件编码异常，尝试不同编码重新读取
4. 将分析结果返回

### 场景二：写入文件
1. 确认目标目录是否存在，必要时使用 `list_directory` 检查
2. 使用 `write_file` 工具将内容写入指定路径
3. 使用 `read_file` 工具验证写入结果
4. 返回写入结果确认

## 限制条件
- 不会删除包含重要系统文件的目录
- 写入前会确认目标路径的安全性和合法性
```

```markdown
# 代码审查技能

## 功能概述
对提交的代码进行全面的审查，包括代码风格、逻辑正确性、安全性和性能等方面。

## 适用场景
- 用户提交了一段代码需要审查
- 用户需要代码改进建议
- PR Review 自动化

## 执行步骤

### 场景一：代码审查
1. 仔细阅读用户提供的代码
2. 从以下维度分析代码：
   - 逻辑正确性：逻辑是否有漏洞，边界条件是否处理
   - 安全性：是否存在常见安全漏洞（注入、XSS、权限绕过等）
   - 性能：是否存在明显性能问题
   - 可读性：命名是否清晰，结构是否合理
3. 按严重程度组织审查意见
4. 对每个问题给出具体的改进建议和修改后的代码示例
5. 汇总生成审查报告

## 限制条件
- 仅审查用户提供的代码片段，不假设未提供的上下文
- 不执行或运行代码
```

```markdown
# 数据分析技能

## 功能概述
对 CSV、JSON 等结构化数据文件进行统计分析，包括数据读取、探索性分析、统计计算和结果报告生成。

## 适用场景
- 用户提供了 CSV/JSON 数据文件，需要进行统计分析
- 需要计算数据的均值、中位数、标准差等统计指标
- 需要从数据中提取特定模式或趋势
- 需要生成数据分析报告

## 执行步骤

### 场景一：CSV 数据探索分析
1. 使用 `read_file` 读取 CSV 文件，了解数据结构
2. 使用 `run_command` 执行 `python skills/data-analysis/scripts/describe_csv.py <文件路径>` 获取列统计信息
   （此脚本为定制工具，`graph_agent.tools` 模块中没有 CSV 统计工具）
3. 根据统计信息识别数据特征（缺失值、异常值、分布特征）
4. 使用 `safe_calculator` 进行必要的数值计算
5. 汇总分析结果，生成结构化报告

### 场景二：JSON 数据处理
1. 使用 `read_file` 读取 JSON 文件
2. 解析 JSON 结构，识别嵌套层级和数据类型
3. 根据用户需求提取或聚合数据
4. 使用 `write_file` 保存处理结果
5. 返回处理摘要

## 限制条件
- 数据文件超过 100MB 时提示用户确认
- 不直接修改原始数据文件（只读分析）
- 定制脚本仅存放在 `skills/data-analysis/scripts/` 目录下
```

## 6. 关键设计决策

### 6.1 为什么选择 Markdown 而非纯 YAML？

SKILL.md 采用 "YAML frontmatter + Markdown 正文" 的混合格式：

- **Frontmatter**：承载机器可解析的结构化元数据（名称、工具声明、参数配置），便于 `SkillParser` 自动提取
- **Markdown 正文**：承载面向 LLM 的自然语言指令（功能说明、执行步骤），LLM 原生理解 Markdown 格式，无需额外解析

纯 YAML 的问题在于多段落、多列表的"执行步骤"难以阅读和维护；纯 Markdown 的问题在于缺少结构化元数据，解析器难以提取工具声明。

### 6.2 为什么 Skill 与 SubAgent 是 1:1 映射？

当前设计中，每个 Skill 对应一个 SubAgent。理由：

1. **关注点分离**：每个 Skill 有独立的执行步骤、工具集和限制条件，对应一个专门的 SubAgent 执行者，职责清晰
2. **提示词隔离**：每个 SubAgent 的系统提示词由对应 SKILL.md 生成，不同 Skill 的指令不会互相干扰
3. **匹配简化**：PlanAgent 通过 Skill 名称即可精确匹配 SubAgent，无需处理"一个 Agent 具备多种组合技能"的歧义

未来可扩展为 N:1 映射（一个 SubAgent 具备多个 Skill），但这需要更复杂的提示词编排和任务路由逻辑，当前阶段保持简单。

### 6.3 为什么 `max_iterations` 放在 Skill 层面而非全局？

不同 Skill 的复杂度差异很大：

- `time_query`：获取当前时间，通常 1 次迭代即可完成
- `data-analysis`：可能需要读取文件 → 执行脚本 → 计算 → 写报告，需要多轮迭代

将 `max_iterations` 放在 Skill 定义中，允许每个 Skill 根据自己的复杂度设置合理的迭代上限，避免简单任务浪费 token 或复杂任务过早终止。

### 6.4 为什么分为内置 Skill 和用户自定义 Skill？

将 Skill 分为两种类型基于以下考量：

1. **降低使用门槛**：内置 Skill 随包分发，用户安装 GraphAgent 后即拥有基础能力（文件操作、计算、时间查询等），无需手动创建任何 Skill
2. **关注点分离**：内置 Skill 覆盖通用的、稳定的基础能力，其工具均在 `graph_agent.tools` 模块中有配套实现；用户自定义 Skill 覆盖特定领域的、定制化的需求，可通过 `scripts/` 灵活扩展
3. **维护成本**：内置 Skill 随 GraphAgent 版本升级自动更新，用户无需关心；用户自定义 Skill 由用户自行维护和版本管理
4. **安全边界**：内置 Skill 的工具经过审查，限制在安全范围内；用户自定义 Skill 的脚本由用户自行承担安全责任
5. **覆盖机制**：允许用户自定义 Skill 覆盖同名的内置 Skill，既保留了内置 Skill 的开箱即用体验，又提供了定制化灵活性

### 6.5 为什么 scripts/ 中的脚本不合并到 tools 模块？

用户自定义 Skill 的 `scripts/` 与 `graph_agent.tools` 模块有明确的边界：

- **tools 模块**：放置通用的、可被多个 Skill 复用的工具，需经过完整的代码审查和测试
- **scripts/ 目录**：放置某个 Skill 专属的、实验性的或一次性的脚本，无需走完整的工具注册流程

当一个 `scripts/` 中的脚本被证明有通用价值时，应将其迁移到 `graph_agent.tools` 模块中成为正式工具。这种"先在 scripts/ 中验证，成熟后升级为 tools 模块工具"的渐进式路径，平衡了灵活性和规范性。

### 6.6 脚本安全模型

`scripts/` 目录中的脚本执行遵循与 `run_command` 工具相同的安全策略（命令白名单 + 沙箱限制）。此外：

- 脚本路径限定在 `skills/{skill_name}/scripts/` 范围内，不允许访问其他 Skill 的脚本
- SubAgent 系统提示词中明确告知脚本目录路径，由 LLM 自行判断是否需要调用
- 脚本不随 Skill 注册自动加载到内存，避免引入依赖污染

## 7. 实现阶段

### 第一阶段：核心框架（1 天）

1. 创建 `src/graph_agent/skill/` 模块目录（`__init__.py`, `models.py`, `parser.py`, `loader.py`, `register.py`）
2. 实现 `models.py` — 数据模型定义（SkillMeta 含 `type` 字段, SkillToolDef, SkillDef 等）
3. 实现 `parser.py` — SKILL.md 解析器（frontmatter + body 分离，`type` 字段校验）
4. 实现 `loader.py` — SkillLoader（扫描内置 .md 文件 + 用户子文件夹，同名覆盖逻辑）
5. 实现 `register.py` — SkillRegister（注册、查找、LLM 摘要生成、to_subagent_configs）
6. 将现有 7 个 SubAgent 转换为内置 Skill（在 `src/graph_agent/skills/` 下创建 `file-management.md`、`calculation.md` 等 .md 文件）

### 第二阶段：系统集成（1 天）

7. 重构 `SubAgentRegistry` — 移除 `config/subagents.yaml` 加载逻辑，改为从 SkillRegister 获取全部配置
8. 实现 `build_skill_system_prompt()` — 根据 SkillDef 动态生成 SubAgent 系统提示词，区分内置/用户两种类型
9. 实现 `register_script_tools()` — 将用户 Skill 的 `scripts/` 脚本动态注册为工具
10. 更新启动流程 — 在编排图构建前初始化 SkillRegister 并注入 SubAgentRegistry
11. 清理：移除 `config/subagents.yaml` 和 `prompts/subagent/` 目录的依赖引用

### 第三阶段：测试与文档（0.5 天）

12. 编写单元测试（parser 各字段解析 / loader 双目录扫描 / register 查找与匹配 / 内置 Skill 和用户 Skill 示例加载）
13. 编写集成测试（Skill → SubAgentConfig → PlanAgent 匹配 → SubAgent 执行）
14. 编写 2-3 个用户自定义 Skill 完整示例（含 `reference/` 和 `scripts/`，放在项目 `skills/` 目录下）

## 8. 扩展方向

- [ ] **语义匹配**：PlanAgent 使用 embedding 对 Skill 描述做语义匹配，替代当前的字符串标签匹配
- [ ] **N:1 Skill 映射**：支持一个 SubAgent 具备多个 Skill，运行时根据任务动态选择对应执行步骤
- [ ] **Skill 市场**：支持从远程仓库（GitHub / 对象存储）下载和安装 Skill
- [ ] **Skill 版本管理**：SKILL.md 中增加 `version` 字段，支持 Skill 升级和回滚
- [ ] **Skill 依赖声明**：Skill 可声明对其他 Skill 的依赖（如 `data-analysis` 依赖 `file-management` 读取数据文件）
- [ ] **Skill 热加载**：监听 `skills/` 目录变化，自动 reload 而不重启服务
- [ ] **Skill 执行沙箱**：scripts/ 中的脚本在独立容器/进程中执行，提供更强的隔离性
