# 8. 会话中断与恢复机制设计

## 1. 概述

### 1.1 背景

当前 GraphAgent 的编排执行是一个"全有或全无"的过程：`graph.ainvoke()` 一旦开始就必须运行到结束，外部无法暂停或中断。如果用户发起了一个复杂任务（如代码分析、多步骤数据处理），在 LLM 调用耗时较长时只能等待，无法中途取消并稍后恢复。

### 1.2 目标

- **主动中断**：用户可在任意安全边界中断正在执行的 Agent 编排流程
- **状态持久化**：中断时将完整的编排状态（含阶段、意图、任务计划、子任务结果）保存为检查点
- **无缝恢复**：用户能在之后恢复会话，编排流程从中断点继续执行
- **协议支持**：通过 ACP 协议暴露中断/恢复/状态查询接口

### 1.3 非目标

- 不支持 LLM 调用中途的强制中断（仅支持节点间安全边界中断）
- 不支持分布式恢复（检查点仅在当前进程内有效）
- 不保证 SubAgent ReAct 循环内单个工具调用的中断精度

## 2. 会话状态机

### 2.1 状态定义

```
                    ┌──────────┐
                    │  IDLE    │ 会话已创建，等待用户输入
                    └────┬─────┘
                         │ send_message
                         ▼
                    ┌──────────┐
              ┌────►│ RUNNING  │ 编排图正在执行
              │     └────┬─────┘
              │          │ interrupt / timeout / error
              │          ▼
              │     ┌──────────┐
              │     │ PAUSED   │ 用户主动中断，等待恢复或放弃
              │     └────┬─────┘
              │          │ resume              
              │          ▼                     
              │     ┌──────────┐               
              └─────┤ RESUMING │ 从检查点重新执行编排图
                    └────┬─────┘
                         │ execution complete
                         ▼
                    ┌──────────┐
                    │ COMPLETED│ 编排执行完成，等待下一轮用户输入
                    └────┬─────┘
                         │ send_message (新一轮)
                         ▼
                    ┌──────────┐
                    │ RUNNING  │
                    └──────────┘
```

### 2.2 会话状态枚举

```python
class SessionStatus(str, Enum):
    IDLE = "idle"             # 等待用户输入
    RUNNING = "running"       # 编排图执行中
    PAUSED = "paused"         # 用户中断，存在检查点
    RESUMING = "resuming"     # 正在从检查点恢复
    COMPLETED = "completed"   # 本轮执行完成
    EXPIRED = "expired"       # 会话超时过期
```

### 2.3 状态转换规则

| 当前状态 | 触发事件 | 目标状态 | 前置条件 |
|---------|---------|---------|---------|
| IDLE | `send_message` | RUNNING | — |
| RUNNING | `interrupt` | PAUSED | 到达安全边界 |
| RUNNING | 执行完成 | COMPLETED | — |
| RUNNING | 超时 | EXPIRED | 超过 session_timeout |
| RUNNING | 执行异常 | IDLE | 错误不可恢复 |
| PAUSED | `resume` | RESUMING | 检查点有效 |
| PAUSED | `send_message` | RUNNING | 放弃旧检查点，重新开始 |
| PAUSED | `delete_session` | — | 删除会话和检查点 |
| RESUMING | 执行完成 | COMPLETED | — |
| RESUMING | `interrupt` | PAUSED | 到达安全边界 |
| COMPLETED | `send_message` | RUNNING | 新一轮对话 |

## 3. 协作式中断机制

### 3.1 安全边界定义

中断仅允许在以下安全边界发生，保证编排状态一致：

```
graph LR
    START --> GuardNode
    GuardNode -->|"☆ 安全边界"| PlanNode
    PlanNode -->|"☆ 安全边界"| SubExecNode
    SubExecNode -->|"☆ 安全边界"| PlanNode
    PlanNode -->|"☆ 安全边界"| GuardNode
    GuardNode --> END
```

即：**每两个 LangGraph 节点之间**。在此边界处：
- 当前节点已完成全部计算
- 编排状态（`OrchestrationState`）已完整更新
- 所有产生的 `ga_messages` 已写入 state
- 下一节点尚未开始

### 3.2 实现模型

```
┌──────────────────────────────────────────────────────┐
│                   ACPServer                          │
│                                                      │
│  execute_turn(session_id, user_input)                │
│    │                                                 │
│    ├─ 1. 设置 session.interrupt_event (asyncio.Event)
│    ├─ 2. 调用 graph.ainvoke(initial_state)          │
│    │     │                                           │
│    │     ├─ guard_node(state)                       │
│    │     │   └─ _check_interrupt(state) ← 检查点 #1 │
│    │     ├─ plan_node(state)                        │
│    │     │   └─ _check_interrupt(state) ← 检查点 #2 │
│    │     ├─ subagent_exec_node(state)               │
│    │     │   └─ _check_interrupt(state) ← 检查点 #3 │
│    │     └─ ...                                     │
│    │                                                 │
│    ├─ 3. 捕获 InterruptException → 保存 checkpoint  │
│    └─ 4. 设置 session.status = PAUSED               │
│                                                      │
│  handle_interrupt(session_id):                       │
│    └─ session.interrupt_event.set()                 │
│                                                      │
│  resume_session(session_id):                         │
│    ├─ 1. 加载 checkpoint state                      │
│    ├─ 2. 调用 graph.ainvoke(checkpoint_state)       │
│    └─ 3. 合并结果                                    │
└──────────────────────────────────────────────────────┘
```

### 3.3 核心代码设计

```python
# 中断信号 — 挂载在 ConversationContext 上
class ConversationContext:
    # ... 现有字段 ...
    interrupt_event: asyncio.Event = field(default_factory=asyncio.Event)
    status: str = "idle"  # SessionStatus 值
    checkpoint: dict | None = None  # 检查点数据

# 中断异常 — 协作式信号
class InterruptException(Exception):
    """编排图节点抛出此异常以响应中断请求。"""
    def __init__(self, state: dict):
        self.state = state  # 携带当前编排状态作为检查点

# 节点内中断检查函数
def _check_interrupt(state: OrchestrationState) -> None:
    """编排图节点在完成核心逻辑后调用此函数检查中断信号。"""
    import asyncio
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        return  # 无事件循环，不在异步上下文中
    
    # 从 state 中获取 interrupt_event（通过 context 传递）
    interrupt_event = state.get("_interrupt_event")
    if interrupt_event and interrupt_event.is_set():
        raise InterruptException(dict(state))
```

### 3.4 节点修改方式

每个编排节点在执行完逻辑后、return 前调用 `_check_interrupt`：

```python
def guard_node(state: OrchestrationState) -> dict:
```

### 3.5 中断响应延迟

| 场景 | 最大延迟 | 说明 |
|------|---------|------|
| 位于 GuardAgent 节点中 | ~5s | 等待当前 LLM 调用完成 |
| 位于 PlanAgent 节点中 | ~10s | 等待当前 LLM 调用完成 |
| 位于 SubAgent 执行中 | ~30s | 等待当前工具调用完成 |
| 位于节点之间 | ~0ms | 立即可中断 |

## 4. 检查点设计

### 4.1 检查点数据结构

```python
@dataclass
class SessionCheckpoint:
    """一次中断的完整快照。"""
    
    # 编排状态
    phase: str                        # 当前编排阶段
    intent: str                       # 用户意图
    task_plan: dict | None            # 任务计划（序列化为 dict）
    sub_results: dict[str, str]       # 已完成的子任务结果
    plan_approved: bool               # 方案是否已审核通过
    result_approved: bool             # 结果是否已审核通过
    review_retries: int               # 已重试次数
    final_answer: str                 # 已生成的最终回复
    
    # 上下文
    messages: list[dict]              # 编排图 messages 通道（langchain 消息序列化）
    ga_messages: list[dict]           # Agent 消息通道
    
    # 元信息
    session_id: str
    turn_index: int                   # 当前对话轮次
    created_at: str                   # 检查点创建时间
    reason: str                       # 中断原因：interrupt / timeout / error
    
    # 恢复信息
    recovery_hint: str                # 恢复提示文本
    node_history: list[str]           # 已执行的节点路径，如 ["guard", "plan", "sub_exec"]
```

```python
def serialize_checkpoint(state: dict, session_id: str, reason: str) -> dict:
    """将 OrchestrationState 序列化为可持久化的检查点。"""
    
    # 序列化 task_plan（dataclass → dict）
    task_plan = state.get("task_plan")
    task_plan_dict = None
    if task_plan:
        task_plan_dict = {
            "plan_id": task_plan.plan_id,
            "overall_goal": task_plan.overall_goal,
            "sub_tasks": [
                {
                    "task_id": t.task_id,
                    "description": t.description,
                    "required_skill": t.required_skill,
                    "assigned_agent": t.assigned_agent,
                    "input_data": t.input_data,
                    "dependencies": t.dependencies,
                    "status": t.status,
                    "result": t.result,
                    "error": t.error,
                }
                for t in task_plan.sub_tasks
            ],
            "execution_strategy": task_plan.execution_strategy,
            "expected_output_format": task_plan.expected_output_format,
        }
    
    # 序列化 messages（LangChain 消息 → dict）
    from langchain_core.messages import message_to_dict
    messages = state.get("messages", [])
    serialized_messages = [message_to_dict(m) for m in messages]
    
    # 序列化 ga_messages（MessageBlock → dict）
    from graph_agent.session.persistence import _serialize_message
    ga_msgs = state.get("ga_messages", [])
    serialized_ga = [
        _serialize_message(m) if hasattr(m, 'role') else str(m) 
        for m in ga_msgs
    ]
    
    return {
        "phase": str(state.get("phase", "").value) if hasattr(state.get("phase", ""), "value") else str(state.get("phase", "")),
        "intent": state.get("intent", ""),
        "task_plan": task_plan_dict,
        "sub_results": state.get("sub_results", {}),
        "plan_approved": state.get("plan_approved", False),
        "result_approved": state.get("result_approved", False),
        "review_retries": state.get("review_retries", 0),
        "final_answer": state.get("final_answer", ""),
        "messages": serialized_messages,
        "ga_messages": serialized_ga,
        "session_id": session_id,
        "turn_index": state.get("turn_index", 0),
        "created_at": _iso_now(),
        "reason": reason,
    }
```

```python
def generate_recovery_hint(checkpoint: dict) -> str:
    """根据检查点状态生成用户可读的恢复提示。"""
    phase = checkpoint.get("phase", "")
    plan = checkpoint.get("task_plan")
    sub_results = checkpoint.get("sub_results", {})
    
    if not plan:
        return "任务尚未制定计划"
    
    total_tasks = len(plan.get("sub_tasks", []))
    completed = len(sub_results)
    
    hints = {
        "intent_analysis": "正在分析您的意图，尚未开始执行",
        "plan_generation": "正在制定任务计划",
        "plan_review": "任务计划已制定，等待审核",
        "task_execution": f"任务已执行 {completed}/{total_tasks} 个子任务，恢复后将继续执行剩余子任务",
        "result_synthesis": "子任务已全部执行完毕，正在汇总结果",
        "result_review": "结果汇总已完成，等待审核",
        "completed": "任务已全部完成",
    }
    return hints.get(phase, f"编排阶段: {phase}")
```

## 5. ACPServer 执行流程改造

### 5.1 execute_turn 改造

```
当前流程:
    1. 压缩检查
    2. 准备上下文
    3. 安装 tracer hook
    4. graph.ainvoke()  ← 不可中断
    5. 收集事件
    6. 持久化
    7. 返回事件列表

改造后流程:
    1. 压缩检查
    2. 准备上下文
    3. 检查是否存在未完成的 checkpoint
       ├─ 存在 → 从 checkpoint 恢复 state，进入 RESUMING 模式
       └─ 不存在 → 正常开始，进入 RUNNING 模式
    4. 安装 tracer hook
    5. 注入 interrupt_event 到 state
    6. graph.ainvoke() 
       └─ try/except InterruptException:
            ├─ 保存 checkpoint → 设置 PAUSED → 返回 pause 事件
            └─ 其他异常 → 清理 checkpoint → 返回 error 事件
    7. 收集事件
    8. 清理 checkpoint（执行成功完成）
    9. 持久化
    10. 返回事件列表
```

```python
async def execute_turn(self, session_id: str, user_input: str) -> list[ACPMessage]:
    events: list[ACPMessage] = []
    ctx = self._session_manager.get_context(session_id)
    if ctx is None:
        return [ACPMessage.error(ErrorCode.SESSION_NOT_FOUND, ...)]
    
    # 状态检查
    if ctx.status == SessionStatus.RUNNING:
        return [ACPMessage.error(ErrorCode.SESSION_LIMIT, "会话正在执行中")]
        
    ctx.status = SessionStatus.RUNNING
    ctx.interrupt_event.clear()
    
    try:
        # 1. 压缩检查
        comp_result = self._session_manager.compress_if_needed(session_id)
        if comp_result.did_compress:
            events.append(...)
        
        # 2. 准备上下文 (含 checkpoint 恢复)
        if ctx.checkpoint:
            # 从 checkpoint 恢复
            initial_state = _deserialize_checkpoint(ctx.checkpoint)
            events.append(ACPMessage.event(PushEvent.PHASE_CHANGED, {
                "phase": "resume",
                "detail": f"从检查点恢复，阶段: {initial_state.get('phase', '')}",
            }))
        else:
            ctx.history.add_user_message(user_input)
            context_msgs = ctx.history.get_context_messages()
            context_lc = agent_messages_to_langchain(context_msgs)
            context_lc.append(HumanMessage(content=user_input))
            initial_state = {"messages": context_lc}
        
        # 3. 注入中断控制
        initial_state["_interrupt_event"] = ctx.interrupt_event
        
        # 4. 安装 tracer hook
        collector = ACPEventCollector(session_id)
        hook = _ACPTracerHook(collector, self._tracer)
        hook.install()
        
        # 5. 执行编排图
        try:
            result = await self._graph.ainvoke(initial_state)
        except InterruptException as e:
            # 保存检查点
            checkpoint = serialize_checkpoint(e.state, session_id, "interrupt")
            ctx.checkpoint = checkpoint
            ctx.status = SessionStatus.PAUSED
            self._session_manager.save_session(session_id)
            
            events.extend(collector.drain())
            events.append(ACPMessage.event(PushEvent.EXECUTION_PAUSED, {
                "session_id": session_id,
                "phase": checkpoint["phase"],
                "recovery_hint": generate_recovery_hint(checkpoint),
            }))
            return events
        
        # 6. 正常完成
        ctx.checkpoint = None  # 清理检查点
        ctx.status = SessionStatus.COMPLETED
        
        events.extend(collector.drain())
        ga_msgs = result.get("ga_messages", [])
        if ga_msgs:
            ctx.history.add_agent_messages(list(ga_msgs))
        
        final_answer = result.get("final_answer", "")
        if not final_answer and result.get("messages"):
            final_answer = result["messages"][-1].content
        ctx.history.add_final_answer(final_answer)
        
        events.append(ACPMessage.event(PushEvent.FINAL_ANSWER, {"content": final_answer}))
        events.append(ACPMessage.event(PushEvent.EXECUTION_COMPLETE, {...}))
        
        self._session_manager.save_session(session_id)
        
    except Exception as e:
        ctx.status = SessionStatus.IDLE
        ctx.checkpoint = None  # 错误不可恢复
        events.append(ACPMessage.error(ErrorCode.EXECUTION_ERROR, str(e)))
    finally:
        hook.uninstall()
        events.extend(collector.drain())
    
    ctx.touch()
    return events
```

### 5.2 handle_interrupt 改造

```python
def _handle_interrupt(self, request: ACPMessage) -> ACPMessage:
    """处理中断请求：设置 interrupt_event，让运行中的编排图在下个安全边界暂停。"""
    session_id = request.payload.get("session_id", "")
    if not session_id:
        return ACPMessage.error(ErrorCode.INVALID_REQUEST, "缺少 session_id")
    
    ctx = self._session_manager.get_context(session_id)
    if ctx is None:
        return ACPMessage.error(ErrorCode.SESSION_NOT_FOUND, f"会话不存在: {session_id}")
    
    if ctx.status != SessionStatus.RUNNING:
        return ACPMessage.error(
            ErrorCode.INVALID_REQUEST,
            f"会话状态为 {ctx.status.value}，无法中断",
        )
    
    # 设置中断事件，编排图将在下一个安全边界响应
    ctx.interrupt_event.set()
    
    return ACPMessage.response(ResponseEvent.ACK, {
        "request_id": request.id,
        "session_id": session_id,
        "status": "interrupt_requested",
        "message": "中断信号已发送，等待编排图到达安全边界",
    })
```

## 7. 超时与异常处理

### 7.1 执行超时

```python
# ACPServer.execute_turn 中添加超时控制
async def execute_turn(self, session_id: str, user_input: str) -> list[ACPMessage]:
    timeout = self._config.execution_timeout  # 新增配置项，默认 300s
    
    try:
        result = await asyncio.wait_for(
            self._graph.ainvoke(initial_state),
            timeout=timeout,
        )
    except asyncio.TimeoutError:
        # 超时视为可恢复中断
        checkpoint = serialize_checkpoint(
            _extract_current_state(), session_id, "timeout"
        )
        ctx.checkpoint = checkpoint
        ctx.status = SessionStatus.PAUSED
        return [ACPMessage.event(PushEvent.EXECUTION_PAUSED, {
            "session_id": session_id,
            "reason": "timeout",
            "recovery_hint": "执行超时，已自动保存检查点",
        })]
```

### 7.2 LLM 调用异常

| 异常类型 | 处理策略 | 会话状态 |
|---------|---------|---------|
| 网络超时 | 自动重试 2 次（间隔 2s/4s），失败后保存检查点 | PAUSED |
| API 限流 (429) | 自动重试 3 次（间隔 5s/10s/20s），失败后保存检查点 | PAUSED |
| 模型不可用 (5xx) | 立即保存检查点，等待恢复 | PAUSED |
| Token 超限 | 触发压缩后重试，仍失败则标记错误 | IDLE |
| 内容安全过滤 | 不重试，直接返回错误 | IDLE |

### 7.3 会话超时

```python
class SessionConfig:
    # 新增配置
    checkpoint_ttl_minutes: int = 1440  # 检查点保留 24 小时
    max_pause_duration_minutes: int = 60  # 最大暂停时长，超时自动标记为过期
    
class SessionManager:
    def cleanup_expired(self) -> int:
        """清理过期会话和过期检查点。"""
        now = datetime.now(timezone.utc)
        cleaned = 0
        
        for sid, ctx in list(self._active.items()):
            if ctx.status == SessionStatus.PAUSED:
                checkpoint_time = datetime.fromisoformat(
                    ctx.checkpoint.get("created_at", "")
                )
                if (now - checkpoint_time).total_seconds() > self._config.max_pause_duration * 60:
                    ctx.status = SessionStatus.EXPIRED
                    ctx.checkpoint = None
                    self.save_session(sid)
                    del self._active[sid]
                    cleaned += 1
            elif self._is_session_expired(ctx):
                self.save_session(sid)
                del self._active[sid]
                cleaned += 1
                
        return cleaned
```

## 10. 风险与约束

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| LLM 调用中无法中断 | 用户等待时间最长 30s+ | 前端显示"等待安全边界"倒计时 |
| 检查点数据随执行进度累积变大 | 存储和传输压力 | 复用现有压缩机制，限制检查点最大 1MB |
| 恢复时编排状态与 LLM 状态不一致 | 恢复后执行结果可能与预期不同 | 恢复时在 system prompt 中注入"从检查点恢复"上下文 |
| 多次中断/恢复导致 auth token 过期 | 恢复时 LLM 调用失败 | 在恢复前检查 API key 有效性 |
| LangGraph 版本升级可能改变 state 结构 | 检查点反序列化失败 | 在检查点中记录 langgraph 版本号 |
