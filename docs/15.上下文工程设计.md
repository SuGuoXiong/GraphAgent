# GraphAgent 上下文工程设计文档

## 1. 概述

GraphAgent 的上下文管理采用**四层分层架构**，每层在上层基础上做减法，向下游 Agent 提供逐步精简的上下文视图。核心原则是：**每个 Agent 只看到完成其职责所需的最小上下文**，避免无关信息稀释 LLM 注意力。

```
┌─────────────────────────────────────────────────────────────────┐
│                      Layer 1: 持久化存储                          │
│              全量消息（含审核/派发/工具调用），磁盘持久化               │
│              职责：完整可追溯，问题排查，审计日志                      │
├─────────────────────────────────────────────────────────────────┤
│                      Layer 2: GuardAgent 战略视图                  │
│              过滤内部消息 + 压缩 + 保留关键语义                        │
│              职责：意图识别、方案审核、结果验收                         │
│              [扩展点] 用户长期偏好                                   │
├─────────────────────────────────────────────────────────────────┤
│                      Layer 3: PlanAgent 战术视图                    │
│              Layer 2 + GuardAgent 意图分析结果                       │
│              职责：任务分解、方案制定、结果汇总                         │
│              [扩展点] Agent 记忆检索                                 │
├─────────────────────────────────────────────────────────────────┤
│                      Layer 4: SubAgent 执行视图                     │
│              仅包含完成任务所需的最小上下文                             │
│              职责：执行具体子任务（工具调用、数据处理等）                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.1 设计动机

当前系统中，所有编排节点（GuardAgent / PlanAgent / SubAgent）共享同一个 `state["messages"]`。这带来三个问题：

1. **上下文膨胀**：编排流程每轮产生 8+ 条内部消息，全部注入 `state["messages"]`，后续节点被迫消费冗余信息
2. **注意力稀释**：SubAgent 执行时收到完整的对话历史（含审核消息、PlanAgent 方案等），但其任务仅需知道"做什么"和"依赖谁的结果"
3. **职责模糊**：各层 Agent 自行调用 `_format_context()` 做硬编码截断（最后 10 条，每条约 500 字符），缺乏统一的分层策略

四层架构将上下文工程从"各 Agent 自行裁剪"升级为"系统级分层注入"，每层 Agent 接收经过针对性精简的上下文。

### 1.2 与现有系统的关系

本文档是对 `docs/6.多轮对话系统设计.md` 的升级和重构。现有系统中的以下模块直接映射到新架构：

| 现有模块 | 映射到新架构 |
|---------|------------|
| `ConversationHistory.messages` | Layer 1（持久化存储） |
| `get_context_messages()` + 压缩器 | Layer 2（GuardAgent 战略视图） |
| `state["messages"]` + `_format_context()` | 重构为 Layer 2/3/4 分层注入 |
| SubAgent `task_description` | 重构为 Layer 4（SubAgent 执行视图） |

## 2. 四层架构详解

### 2.1 Layer 1 —— 持久化存储

**定位**：最底层，保存每次对话中所有 Agent 产生的全部消息，不做任何裁剪。

**数据来源**：
- 用户输入（`USER_INPUT`）
- GuardAgent 输出（`GUARD_INTENT_ANALYSIS`、`GUARD_PLAN_REVIEW`、`GUARD_RESULT_REVIEW`）
- PlanAgent 输出（`PLAN_PROPOSAL`、`PLAN_TASK_DISPATCH`、`PLAN_RESULT_SYNTHESIS`）
- SubAgent 输出（`SUBAGENT_TASK_RESULT`）
- 工具调用记录（`TOOL_CALL`、`TOOL_RESULT`）
- 最终回复（`FINAL_ANSWER`）
- 系统消息（`SYSTEM_PROMPT`、`SYSTEM_NOTIFICATION`）

**存储形式**：`ConversationHistory.messages: list[MessageBlock]`

**职责**：
- 完整可追溯 —— 磁盘上的 JSON 文件包含每轮对话的全部编排流程记录
- 问题排查 —— 出现异常时可回溯完整的 Agent 交互链
- 审计日志 —— 审核消息（`GUARD_PLAN_REVIEW` / `GUARD_RESULT_REVIEW`）的 JSON 内容被完整保留

**实现**：沿用现有 `ConversationHistory` + `ConversationPersistence`，无需修改。

```python
# Layer 1 接口（现有实现，保持不变）
class ConversationHistory:
    session_id: str
    messages: list[MessageBlock]   # 全量，永不裁剪
    turn_count: int

    def add_user_message(self, content: str) -> MessageBlock: ...
    def add_agent_messages(self, messages: list[MessageBlock]) -> None: ...
    def add_final_answer(self, content: str) -> None: ...
```

**数据流出口**：作为 Layer 2 的输入源。

### 2.2 Layer 2 —— GuardAgent 战略视图

**定位**：在 Layer 1 基础上过滤掉内部编排产物，在达到压缩阈值时执行压缩。为 GuardAgent（意图识别、方案审核、结果验收）提供战略级上下文。

**数据来源**：Layer 1 的全量消息。

**处理步骤**：

#### 2.2.1 上下文过滤

沿用现有 `ContextFilter` 规则（`docs/6.多轮对话系统设计.md` 第 6 节），排除以下消息类型：

| 排除的消息类型 | 理由 |
|--------------|------|
| `GUARD_PLAN_REVIEW` | 纯审核 JSON，无语义价值 |
| `GUARD_RESULT_REVIEW` | 纯审核 JSON，无语义价值 |
| `PLAN_TASK_DISPATCH` | 任务派发指令，语义已被 SubAgent 结果覆盖 |
| `TOOL_CALL` | 工具调用细节，语义已被 SubAgent 结果覆盖 |
| `TOOL_RESULT` | 工具调用细节，语义已被 SubAgent 结果覆盖 |

此外，新增过滤规则：**被驳回的中间产物**——未通过 GuardAgent 审核的 `PLAN_PROPOSAL` 和 `PLAN_RESULT_SYNTHESIS` 应当排除。这类消息是已被重新生成的过期版本，保留在上下文中只会造成混淆。

**实现方式**：在 `MessageBlock.metadata` 中增加 `approved` 标记，过滤时检查。

```python
# 新增过滤逻辑：排除被驳回的中间产物
def _is_rejected_proposal(msg: MessageBlock) -> bool:
    """检查是否为被驳回的方案/汇总消息。"""
    msg_type = MessageType(msg.message_type)
    if msg_type not in (MessageType.PLAN_PROPOSAL, MessageType.PLAN_RESULT_SYNTHESIS):
        return False
    return msg.metadata.get("approved") is False
```

#### 2.2.2 自适应压缩

沿用现有两级压缩机制（`docs/6.多轮对话系统设计.md` 第 7 节）：

```
总 Token 占比        触发动作
────────────────────────────────────────────
< 60%                不压缩
60% ~ 80%            普通压缩：按优先级 P4→P1 删除低价值消息
> 80%                高度压缩：LLM 摘要替换历史消息
```

压缩保护窗口：最近 `keep_recent_turns` 轮对话不参与压缩。

### 2.3 Layer 3 —— PlanAgent 战术视图

**定位**：在 Layer 2 基础上增加 GuardAgent 意图识别结果。为 PlanAgent（任务分解、方案制定、结果汇总）提供战术级上下文。

**数据来源**：Layer 2 过滤压缩后的消息 + GuardAgent 意图分析结果。

**核心区别**：Layer 2 为 GuardAgent 服务，Layer 3 为 PlanAgent 服务。两者的关键差异在于 Layer 3 额外包含：

1. **GuardAgent 意图分析结果**（`GUARD_INTENT_ANALYSIS`）：提炼后的用户意图、问题类型、复杂度评估。PlanAgent 依赖此结果做任务分解
2. **可用的 SubAgent 清单**：以结构化形式注入，供 PlanAgent 匹配技能

**处理步骤**：

#### 2.3.1 上下文构建

```
Layer 2（GuardAgent 战略视图）
    +
GuardAgent 意图分析结果（当前轮次的 GUARD_INTENT_ANALYSIS）
    +
SubAgent 能力清单（结构化描述）
    =
Layer 3（PlanAgent 战术视图）
```

注意：PlanAgent 不需要看到 `GUARD_PLAN_REVIEW` 和 `GUARD_RESULT_REVIEW`，这些已在 Layer 2 中被过滤。

- 失败经验（"上次用 SubAgent X 处理 Y 类任务时结果被驳回，建议改用 SubAgent Z"）
- 用户对历史方案的反馈（"用户偏好并行执行而非串行"）

### 2.4 Layer 4 —— SubAgent 执行视图

**定位**：PlanAgent 完成任务分解后，为每个 SubAgent 注入完成任务所需的**最小上下文**。这是四层架构中裁剪最激进的一层。

**设计原则**：SubAgent 是执行者，不是决策者。它不需要了解完整的对话历史、不需要看到 GuardAgent 的审核过程、不需要知道其他并行 SubAgent 在做什么。它只需要知道：**做什么、用什么做、依赖谁的结果**。

**每个 SubAgent 接收的上下文包含三部分**：

#### 2.4.1 任务详细信息（Task Detail）

- **任务描述** — PlanAgent 分解的具体任务文本（`SubTask.description`）
- **输入参数** — 解析占位符后的 `SubTask.input_data`
- **期望产出** — 任务计划中声明的 `expected_output_format`

#### 2.4.2 依赖的前置输入（Dependency Inputs）

- **前置任务结果** — 当前任务 `dependencies` 列表中所有已完成 SubAgent 的执行结果
- **数据流转标注** — 标明每个前置结果的来源（哪个 SubAgent 产出）和用途（当前任务的哪个参数引用它）

```python
def _format_dependency_inputs(
    task: SubTask,
    all_tasks: list[SubTask],
) -> str:
    """格式化依赖的前置输入。

    为每个依赖项标注来源 Agent 和结果内容，让 SubAgent 理解数据血缘。
    """
    if not task.dependencies:
        return "(无依赖输入)"

    parts = []
    for dep_id in task.dependencies:
        dep_task = next((t for t in all_tasks if t.task_id == dep_id), None)
        if dep_task is None:
            parts.append(f"- [{dep_id}]: (任务不存在)")
            continue
        if dep_task.status != "completed":
            parts.append(f"- [{dep_id}]: (任务状态: {dep_task.status})")
            continue

        agent_name = dep_task.assigned_agent or "unknown"
        result_preview = dep_task.result[:2000] if dep_task.result else "(无结果)"
        parts.append(
            f"### 前置任务 [{dep_id}] (执行者: {agent_name})\n"
            f"**任务描述**: {dep_task.description[:300]}\n"
            f"**执行结果**:\n{result_preview}"
        )
    return "\n\n".join(parts)
```

#### 2.4.3 任务背景的关键描述（Task Background）

- **用户原始意图摘要** — 从 Layer 3 的 `GUARD_INTENT_ANALYSIS` 中提取的一句话意图描述
- **整体目标** — 任务计划的 `overall_goal`，让 SubAgent 理解其工作在全局中的位置
- **上下文片段** — 从 Layer 2 中提取与当前子任务直接相关的历史消息（如用户提供的文件路径、参数值等）

```python
def _extract_relevant_context(
    guard_context: list[MessageBlock],
    task: SubTask,
    max_tokens: int = 1000,
) -> str:
    """从战略视图中提取与当前任务相关的上下文片段。

    策略：扫描 guard_context 中的 USER_INPUT 和 FINAL_ANSWER 消息，
    提取包含任务关键词的片段。
    """
    # 提取任务中的关键词
    keywords = set()
    for word in task.description.split():
        if len(word) >= 2:
            keywords.add(word.lower())

    relevant_parts = []
    token_budget = max_tokens
    for msg in guard_context:
        msg_type = MessageType(msg.message_type)
        if msg_type not in (MessageType.USER_INPUT, MessageType.FINAL_ANSWER):
            continue
        content = msg.content if isinstance(msg.content, str) else ""
        if not content:
            continue
        # 简单相关度判断：消息内容与任务关键词的交集
        overlap = sum(1 for kw in keywords if kw in content.lower())
        if overlap >= 2:
            snippet = content[:500]
            relevant_parts.append(f"[{msg_type.value}]: {snippet}")
            token_budget -= len(snippet) // 2  # 粗略估算
            if token_budget <= 0:
                break

    return "\n---\n".join(relevant_parts) if relevant_parts else "(无特别相关上下文)"
```

#### 2.4.4 上下文组装

```python
def build_subagent_context(
    task: SubTask,
    all_tasks: list[SubTask],
    overall_goal: str,
    intent_summary: str,
    guard_context: list[MessageBlock],
    expected_output_format: str = "",
) -> str:
    """构建单个 SubAgent 的执行视图上下文。

    组装四部分内容：
    1. 任务背景
    2. 任务详细信息
    3. 依赖的前置输入
    4. 相关上下文片段
    """
    parts = []

    # 1. 任务背景
    parts.append("## 任务背景")
    parts.append(f"**用户意图**: {intent_summary}")
    parts.append(f"**整体目标**: {overall_goal}")
    if expected_output_format:
        parts.append(f"**期望产出格式**: {expected_output_format}")

    # 2. 任务详细信息
    parts.append("\n## 当前任务")
    parts.append(f"**任务ID**: {task.task_id}")
    parts.append(f"**所需技能**: {task.required_skill}")
    parts.append(f"**任务描述**:\n{task.description}")

    # 3. 依赖的前置输入
    parts.append("\n## 前置任务输出")
    parts.append(_format_dependency_inputs(task, all_tasks))

    # 4. 相关上下文
    parts.append("\n## 相关上下文")
    parts.append(_extract_relevant_context(guard_context, task))

    return "\n".join(parts)
```

## 3. 编排流程集成

### 3.1 整体数据流

```
                          ┌──────────────────────┐
                          │  ConversationHistory  │
                          │  (Layer 1: 全量存储)   │
                          └──────────┬───────────┘
                                     │
                          ┌──────────▼───────────┐
                          │  GuardContextBuilder  │
                          │  (Layer 2: 过滤+压缩)  │
                          └──────────┬───────────┘
                                     │
                          ┌──────────▼───────────┐
                          │  PlanContextBuilder   │
                          │  +intent +subagents   │
                          │  (Layer 3: 战术视图)   │
                          └──────────┬───────────┘
                                     │
                          ┌──────────▼───────────┐
                          │ SubAgentContextBuilder│
                          │   per-task minimal    │
                          │  (Layer 4: 执行视图)   │
                          └──────────────────────┘
```

### 3.2 编排流程中的调用时序

```
1. 用户输入到达
2. Layer 2 构建: GuardContextBuilder.build(history)
   → guard_context: list[MessageBlock]
3. Layer 2 上下文注入 GuardAgent 节点
   → GuardAgent 执行意图分析，产出 intent_analysis
4. Layer 3 构建: PlanContextBuilder.build(history, intent_analysis, registry)
   → plan_context: list[MessageBlock]
5. Layer 3 上下文注入 PlanAgent 节点
   → PlanAgent 执行任务分解，产出 TaskPlan
6. Layer 4 构建: SubAgentContextBuilder.build_all(task_plan, intent_analysis, guard_context)
   → subagent_contexts: dict[task_id, str]
7. Layer 4 上下文注入各 SubAgent 节点
   → SubAgent 并行执行
8. PlanAgent 结果汇总（使用 Layer 3 上下文）
9. GuardAgent 结果验收（使用 Layer 2 上下文）
10. 持久化 Layer 1
```

```python
def build_orchestration_graph():
    builder = StateGraph(OrchestrationState)

    builder.add_node("guard", guard_node)
    builder.add_node("plan", plan_node)
    builder.add_node("sub_exec", subagent_exec_node)

    # 新增：上下文构建节点

    builder.add_node("build_subagent_contexts", build_subagent_contexts_node)  # Layer 4

    # 新流程: START → build_guard_context → guard → build_plan_context → plan
    #         → build_subagent_contexts → sub_exec → plan → guard → END
    builder.add_edge(START, "build_guard_context")
    builder.add_edge("build_guard_context", "guard")
    builder.add_conditional_edges("guard", guard_router, {
        "plan": "build_plan_context",  # 改为先构建 Layer 3 再进入 PlanAgent
        "__end__": END,
    })
    builder.add_edge("build_plan_context", "plan")
    builder.add_conditional_edges("plan", plan_router, {
        "guard": "guard",
        "sub_exec": "build_subagent_contexts",  # 改为先构建 Layer 4 再执行 SubAgent
        "plan": "plan",
        "__end__": END,
    })
    builder.add_edge("build_subagent_contexts", "sub_exec")
    builder.add_edge("sub_exec", "plan")

    return builder.compile()
```

**上下文构建节点实现**：

```python
def build_guard_context_node(state: OrchestrationState) -> dict:
    """Layer 2 上下文构建节点 —— 在 GuardAgent 执行前调用。"""
    from graph_agent.acp.session_manager import _session_manager
    from graph_agent.llm import LLMFactory

    session_id = state.get("_session_id", "")
    ctx = _session_manager.get_context(session_id)
    if ctx is None:
        return {}

    builder = GuardContextBuilder(SessionConfig.from_yaml())
    provider = LLMFactory.create_from_env()
    guard_context = builder.build(
        ctx.history,
        llm_provider=provider,
        user_id=state.get("user_id"),
    )

    # 同时复制到 state["messages"] 以保持兼容
    from graph_agent.message import agent_messages_to_langchain
    return {
        "guard_context": guard_context,
        "messages": agent_messages_to_langchain(guard_context),
    }


def build_plan_context_node(state: OrchestrationState) -> dict:
    """Layer 3 上下文构建节点 —— 在 PlanAgent 执行前调用。"""
    from graph_agent.acp.session_manager import _session_manager

    session_id = state.get("_session_id", "")
    ctx = _session_manager.get_context(session_id)
    if ctx is None:
        return {}

    # 获取 GuardAgent 最新的意图分析结果
    ga_messages = state.get("ga_messages", [])
    intent_analysis = next(
        (m for m in ga_messages
         if MessageType(m.message_type) == MessageType.GUARD_INTENT_ANALYSIS),
        None,
    )

    builder = PlanContextBuilder(SessionConfig.from_yaml())
    plan_context = builder.build(
        ctx.history,
        intent_analysis=intent_analysis,
        subagent_registry=_registry,
        user_id=state.get("user_id"),
    )

    from graph_agent.message import agent_messages_to_langchain
    return {
        "plan_context": plan_context,
        "messages": agent_messages_to_langchain(plan_context),
    }


def build_subagent_contexts_node(state: OrchestrationState) -> dict:
    """Layer 4 上下文构建节点 —— 在 SubAgent 执行前调用。"""
    task_plan = state.get("task_plan")
    ga_messages = state.get("ga_messages", [])
    guard_context = state.get("guard_context", [])

    intent_analysis = next(
        (m for m in ga_messages
         if MessageType(m.message_type) == MessageType.GUARD_INTENT_ANALYSIS),
        None,
    )

    builder = SubAgentContextBuilder()
    subagent_contexts = builder.build_all(
        task_plan=task_plan,
        intent_analysis=intent_analysis,
        guard_context=guard_context,
    )

    return {"subagent_contexts": subagent_contexts}
```

```python
        results = self.search_similar_tasks(intent, top_k)
        if not results:
            return None
        lines = ["## 历史参考案例"]
        for i, r in enumerate(results, 1):
            lines.append(
                f"### 案例 {i} (相似度: {r['score']:.2f})\n"
                f"- 意图: {r['intent']}\n"
                f"- 方案: {r['plan_summary']}\n"
                f"- 结果: {r['outcome']}"
            )
        return create_assistant_message(
            content="\n\n".join(lines),
            message_type=MessageType.AGENT_RESPONSE,
            name="MemoryRetriever",
            metadata={"source": "agent_memory"},
```

**未来实现方向**：
- 向量化：使用 Embedding 模型将意图和方案编码为向量，存储于向量数据库
- 检索策略：语义相似度 + 结果质量加权（成功的方案优先推荐）
- 记忆更新：每轮对话完成后自动记录（`outcome` = GuardAgent 审核结果）

### 6.3 扩展点集成方式

两个扩展点的集成方式一致：在对应 Layer 的 Builder 中持有抽象基类引用，构建上下文时尝试获取扩展信息。当前空壳实现直接返回空结果，不影响主流程。

```python
# GuardContextBuilder.__init__
def __init__(
    self,
    config: SessionConfig,
    preference_store: UserPreferenceStore | None = None,
):
    self._preference_store = preference_store or UserPreferenceStore()  # 默认空壳

# PlanContextBuilder.__init__
def __init__(
    self,
    config: SessionConfig,
    memory_store: AgentMemoryStore | None = None,
):
    self._memory_store = memory_store or AgentMemoryStore()  # 默认空壳
```

这种设计确保：
- **零侵入**：当前不实现扩展功能时，空壳对象直接返回空结果，不影响主流程
- **可替换**：未来只需实现抽象基类的具体子类并注入 Builder 即可激活功能
- **可测试**：单元测试可通过 Mock 注入模拟扩展行为

## 9. 设计优势

1. **注意力聚焦** —— 每层 Agent 只看到完成职责所需的最小上下文，避免注意力被无关信息稀释
2. **渐进裁剪** —— 四层逐级做减法：全量 → 过滤+压缩 → 追加意图 → 最小执行视图
3. **职责清晰** —— GuardAgent 看战略视图（"用户想做什么"），PlanAgent 看战术视图（"怎么分解任务"），SubAgent 看执行视图（"这个任务怎么做"）
4. **可扩展** —— Layer 2/3 预留了用户偏好和记忆检索扩展点，通过抽象基类 + 空壳实现达到零侵入
5. **复用现有资产** —— Layer 1/2 直接复用现有的 `ConversationHistory`、`ContextFilter`、压缩器等模块，不影响已有功能
6. **可降级** —— 扩展点默认空壳实现，不注入偏好/记忆时系统行为不变，未来激活扩展只需替换实现
