# GraphAgent MCP 协议支持设计文档

## 1. 概述

### 1.1 设计背景

GraphAgent 目前仅支持通过内置 `graph_agent.tools` 模块中的工具及用户自定义 Skill 的 `scripts/` 脚本来扩展能力。这两种方式存在以下局限：

- **扩展需修改 GraphAgent 代码或文件系统**：新增工具要么在 `graph_agent/tools/` 中编写 Python 代码，要么在 `skills/*/scripts/` 中放置脚本——都需要直接操作 GraphAgent 运行环境
- **无法复用外部工具生态**：大量已有的 MCP Server（如数据库查询、云服务 API、第三方平台集成等）无法直接接入 GraphAgent
- **跨进程/跨机器通信能力缺失**：当工具逻辑由独立服务提供时（如远端 API 网关、独立微服务），缺乏标准化的集成方式

MCP（Model Context Protocol）是 Anthropic 发布的标准化协议，用于 LLM 应用与外部工具/数据源的通信。通过 MCP，GraphAgent 可以：

- 接入任何兼容 MCP 协议的第三方 Server，无需修改 GraphAgent 自身代码
- 支持本地子进程（stdio）和远程 HTTP（Streamable HTTP）两种通信方式
- 利用 MCP 的工具发现机制（`list_tools`），在启动时自动感知外部 Server 的能力

### 1.2 核心设计理念

```
现有方式（内置工具 + 脚本）:
  graph_agent.tools.*       →  AgentTool  →  ToolCenter
  skills/*/scripts/*.py     →  AgentTool  →  ToolCenter

MCP 方式（外部工具发现）:
  mcp_servers.json          →  MCPManager  →  建立连接  →  list_tools()  →  AgentTool  →  ToolCenter
                                                                              ↑
                                                              MCPToolAdapter（将 MCP Tool 包装为 AgentTool）
```

MCP 工具与内置工具地位平等——统一注册到 `ToolCenter`，对上游编排层（PlanAgent、SubAgent）完全透明。

### 1.3 三个核心功能

| 功能 | 说明 | 对应模块 |
|------|------|---------|
| **MCP 服务发现** | 读取配置文件，识别可用的 MCP Server 及其连接参数 | `mcp/config.py` |
| **MCP 注册** | 与 MCP Server 建立连接，获取工具列表，注册到 ToolCenter | `mcp/manager.py` |
| **MCP 调用** | 运行时通过 ToolCenter 调用 MCP 工具，封装结果为 ToolMessage | `mcp/tool_adapter.py` |

### 1.4 与现有系统的关系

```
                        ┌─────────────────────┐
                        │    ToolCenter        │
                        │  (统一工具注册中心)    │
                        └┬──────┬──────┬──────┘
                         │      │      │
                 ┌───────┘      │      └───────┐
                 ▼              ▼              ▼
          ┌──────────┐  ┌──────────┐  ┌──────────────┐
          │ 内置工具  │  │脚本工具   │  │  MCP 工具     │
          │(tools/)  │  │(scripts) │  │  (外部服务)    │
          └──────────┘  └──────────┘  └──────────────┘
                                         │
                              ┌──────────┴──────────┐
                              ▼                     ▼
                       ┌──────────┐         ┌──────────────┐
                       │ stdio    │         │streamable-http│
                       │(本地进程序)│        │(远程 HTTP)     │
                       └──────────┘         └──────────────┘
```

- **不替代**任何现有工具系统，而是作为第三种工具来源补充到 ToolCenter
- **不影响** Skill 系统、SubAgent 编排层、Hook 机制
- MCP 工具的命名使用 `mcp__<server>__<tool>` 前缀，避免与内置工具名称冲突

### 3.2 config.py —— 配置模型

```python
"""MCP 配置模型——定义 mcp_servers.json 的数据结构。"""

from dataclasses import dataclass, field
from enum import Enum
from typing import Literal
from pathlib import Path


class TransportType(str, Enum):
    STDIO = "stdio"
    STREAMABLE_HTTP = "streamable-http"


@dataclass
class MCPServerConfig:
    """单个 MCP Server 的连接配置。"""
    name: str                              # 服务器逻辑名称（来自 JSON key）
    transport: TransportType

    # stdio 专用参数
    command: str | None = None             # 可执行文件路径
    args: list[str] = field(default_factory=list)
    env: dict[str, str] | None = None
    cwd: str | None = None

    # streamable-http 专用参数
    url: str | None = None
    headers: dict[str, str] | None = None
    timeout: float = 30.0
    sse_read_timeout: float = 300.0

    @classmethod
    def from_dict(cls, name: str, data: dict) -> "MCPServerConfig":
        """从 JSON 字典构造配置对象。"""
        transport = TransportType(data["transport"])
        return cls(
            name=name,
            transport=transport,
            command=data.get("command"),
            args=data.get("args", []),
            env=data.get("env"),
            cwd=data.get("cwd"),
            url=data.get("url"),
            headers=data.get("headers"),
            timeout=data.get("timeout", 30.0),
            sse_read_timeout=data.get("sse_read_timeout", 300.0),
        )


def load_mcp_config(config_path: str | Path = "mcp_servers.json") -> list[MCPServerConfig]:
    """从 JSON 文件加载 MCP 配置列表。

    Args:
        config_path: mcp_servers.json 文件路径，默认为项目根目录下的 mcp_servers.json

    Returns:
        MCPServerConfig 列表
    """
    import json
    config_path = Path(config_path)
    if not config_path.exists():
        return []

    with open(config_path, "r", encoding="utf-8") as f:
        data = json.load(f)

    servers = data.get("mcpServers", {})
    return [MCPServerConfig.from_dict(name, cfg) for name, cfg in servers.items()]
```

**设计要点**：
- 使用 `@dataclass` 而非 Pydantic，与项目现有风格一致（`SubAgentConfig`、`SkillDef` 均为 dataclass）
- `from_dict` 工厂方法封装 JSON → 配置对象的转换
- `load_mcp_config` 是便捷函数，文件不存在时返回空列表（不报错，MCP 为可选功能）
- 两种 transport 的专用参数通过 `| None` 并置默认值来区分，避免定义两个子类（简单优先）

### 3.3 tool_adapter.py —— 工具适配层

```python
"""MCP 工具适配——将 MCP Tool 包装为 AgentTool。"""

from graph_agent.tools.base import AgentTool


def _json_schema_to_params(input_schema: dict) -> list[dict[str, str]]:
    """将 MCP 工具的 JSON Schema inputSchema 转换为 AgentTool 的 parameters 格式。

    JSON Schema 的 properties:
      {"path": {"type": "string", "description": "文件路径"}}

    转换为:
      [{"name": "path", "type": "string", "description": "文件路径"}]
    """
    params = []
    properties = input_schema.get("properties", {})
    required = input_schema.get("required", [])

    for name, schema in properties.items():
        params.append({
            "name": name,
            "type": _json_type_to_str(schema.get("type", "string")),
            "description": schema.get("description", ""),
            "required": name in required,
        })
    return params


def _json_type_to_str(json_type) -> str:
    """将 JSON Schema 类型值转换为字符串表示。"""
    if isinstance(json_type, list):
        return ", ".join(t for t in json_type if t != "null")
    return str(json_type)


def wrap_mcp_tools(mcp_tools: list, server_name: str, session) -> list[AgentTool]:
    """将一批 MCP Tool 对象包装为 AgentTool 列表。

    每个 AgentTool 的命名规则：mcp__<server_name>__<tool_name>

    Args:
        mcp_tools: MCP SDK 返回的 Tool 对象列表
        server_name: MCP Server 的逻辑名称
        session: 对应的 ClientSession 引用，工具调用时需要

    Returns:
        AgentTool 列表
    """
    from graph_agent.mcp.manager import _call_mcp_tool

    agents = []
    for t in mcp_tools:
        tool_name = f"mcp__{server_name}__{t.name}"
        params = _json_schema_to_params(t.inputSchema)
        description = t.description or f"MCP 工具: {t.name} (来自 Server '{server_name}')"

        # 闭包捕获 server_name 和 t.name，运行时调用 MCP session
        def make_func(srv_name: str, mcp_tool_name: str):
            def tool_func(**kwargs) -> str:
                return _call_mcp_tool(srv_name, mcp_tool_name, kwargs)
            return tool_func

        agent = AgentTool(
            name=tool_name,
            description=description,
            func=make_func(server_name, t.name),
            parameters=params,
        )
        agents.append(agent)

    return agents
```

**设计要点**：
- `_json_schema_to_params` 负责 JSON Schema → AgentTool parameters 格式的转换
- 工具名称格式 `mcp__<server>__<tool>` 使用双下划线分隔，简洁且不易与现有工具名冲突
- 每个 AgentTool 的 `func` 是一个闭包，捕获 `server_name` 和 `tool_name`，运行时委托给 MCPManager 的全局 session 执行实际的 `call_tool`
- `_call_mcp_tool` 由 manager.py 提供，是一个模块级函数，通过全局 session 字典查找目标 session

### 3.4 manager.py —— MCP 管理器

```python
"""MCP 管理器——负责 MCP Server 的连接、工具发现和生命周期管理。"""

import asyncio
import threading
from typing import Any

from graph_agent.tools.base import ToolCenter
from graph_agent.tracer import get_tracer


class _SessionContext:
    """封装单个 MCP Server 的 session 和 transport 上下文。

    对于 stdio transport，需要同时持有 transport 的读写流
    以维持子进程的生命周期。对于 HTTP transport，持有 client
    上下文即可（session 存活期间 HTTP 连接池保持）。
    """

    def __init__(self):
        self.session = None          # ClientSession 实例
        self._transport_ctx = None   # transport 上下文管理器（stdio/streamable-http）
        self._read = None            # stdio: MemoryObjectReceiveStream
        self._write = None           # stdio: MemoryObjectSendStream
        self._get_url_fn = None      # streamable-http: get_url 回调


# 全局 session 注册表，供 tool_adapter 的闭包查找
# key: server_name, value: _SessionContext
_sessions: dict[str, _SessionContext] = {}


def _call_mcp_tool(server_name: str, tool_name: str, arguments: dict) -> str:
    """同步调用 MCP 工具，由 AgentTool.func 触发。

    此函数运行在同步上下文中（AgentTool.run 是同步的），
    需要在同步线程内执行异步调用。

    Args:
        server_name: MCP Server 逻辑名称
        tool_name: 原始 MCP 工具名称（不含 mcp__ 前缀）
        arguments: 工具参数字典

    Returns:
        工具执行结果文本
    """
    ctx = _sessions.get(server_name)
    if ctx is None or ctx.session is None:
        return f"错误: MCP Server '{server_name}' 未连接或已断开"

    session = ctx.session

    async def _call():
        return await session.call_tool(tool_name, arguments)

    # 在已有事件循环中运行，或创建新循环
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        return asyncio.run(_call())

    # 当前线程已有事件循环（如被 asyncio.to_thread 包装），创建子线程执行
    result = None
    exc = None

    def _run_in_thread():
        nonlocal result, exc
        try:
            new_loop = asyncio.new_event_loop()
            asyncio.set_event_loop(new_loop)
            result = new_loop.run_until_complete(_call())
        except Exception as e:
            exc = e
        finally:
            new_loop.close()

    thread = threading.Thread(target=_run_in_thread)
    thread.start()
    thread.join()

    if exc:
        return f"MCP 工具调用错误 [{tool_name}]: {exc}"

    return _extract_result_text(result)


def _extract_result_text(result) -> str:
    """从 CallToolResult 中提取文本内容。"""
    if result.isError:
        texts = []
        for c in result.content:
            if hasattr(c, "text"):
                texts.append(f"[ERROR] {c.text}")
        return "\n".join(texts) if texts else "MCP 工具返回错误"

    texts = []
    for c in result.content:
        if hasattr(c, "text"):
            texts.append(c.text)
    return "\n".join(texts) if texts else str(result)
```

**MCPManager 核心逻辑（伪代码 + 关键流程）**：

```
初始化流程:
  1. 调用 load_mcp_config() 读取 mcp_servers.json
  2. 若文件不存在或为空，跳过（MCP 为可选功能）
  3. 对每个 MCPServerConfig：
     a. 根据 transport 类型创建连接
        - stdio: 使用 stdio_client(StdioServerParameters(...))
        - streamable-http: 使用 streamablehttp_client(url, headers, timeout)
     b. 创建 ClientSession(read, write)
     c. 调用 session.initialize()
     d. 调用 session.list_tools() 获取工具列表
     e. 调用 wrap_mcp_tools() 包装为 AgentTool
     f. 调用 tool_center.register() 逐个注册
     g. 保存 session 引用到 _sessions 字典
  4. 记录日志：已连接 N 个 MCP Server，注册 M 个工具
```

**连接生命周期**：

```
                    MCPManager.__init__()
                           │
                           ▼
              ┌────────────────────────┐
              │  load_mcp_config()     │
              │  读取 mcp_servers.json │
              └───────────┬────────────┘
                          │
                    ┌─────┴─────┐
                    ▼           ▼
              ┌──────────┐ ┌──────────────┐
              │  stdio   │ │streamable-http│
              │  连接     │ │  连接         │
              └────┬─────┘ └──────┬───────┘
                   │              │
                   └──────┬───────┘
                          ▼
              ┌────────────────────────┐
              │  session.initialize()  │
              │  session.list_tools()  │
              └───────────┬────────────┘
                          │
                          ▼
              ┌────────────────────────┐
              │  wrap_mcp_tools()       │
              │  tool_center.register() │
              └───────────┬────────────┘
                          │
                          ▼
              ┌────────────────────────┐
              │  保存 session 引用      │
              │  到 _sessions 字典      │
              └────────────────────────┘

运行时工具调用:

   AgentTool.run(kwargs)
        │
        ▼
   _call_mcp_tool(server_name, tool_name, kwargs)
        │
        ▼
   session.call_tool(name, arguments)   ← 异步调用
        │
        ▼
   _extract_result_text(result)         ← 提取文本
```

**关键设计决策**：

1. **全局 `_sessions` 字典**
   - 为什么：`wrap_mcp_tools` 创建 AgentTool 时，每个工具函数的闭包需要持有 session 引用。但 AgentTool 的 `func` 签名是固定的 `(**kwargs) -> str`，不能传递 session 参数。全局字典是最简单的解决方案——MCPManager 在注册阶段将 session 存入，工具函数在调用阶段从中取出。
   - 替代方案评估：如果每个 MCP Server 有独立的 ToolCenter 实例，可以避免全局状态，但这会破坏 ToolCenter 作为"统一注册中心"的设计目标。

2. **同步/异步桥接**
   - MCP SDK 的 `ClientSession.call_tool` 是异步方法
   - `AgentTool.run` 是同步方法（被 LangChain/LangGraph 的同步调用链调用）
   - 桥接策略（按优先级）：
     1. 若当前无运行中的事件循环 → `asyncio.run()` 直接执行（最常见情况）
     2. 若当前有事件循环（如在 async 上下文中被调用）→ 在子线程中创建新事件循环执行

3. **长连接 vs 短连接**
   - 采用**长连接**模式：MCPManager 在 GraphAgent 启动时建立所有连接，保持到进程退出
   - 对于 stdio：子进程在 GraphAgent 整个生命周期内保持运行
   - 对于 streamable-http：HTTP 连接池保持
   - 优势：避免每次工具调用时重复建连/认证的开销

4. **连接失败处理**
   - 单个 MCP Server 连接失败不影响其他 Server，也不影响 GraphAgent 启动
   - 失败的 Server 被跳过，其工具不注册到 ToolCenter
   - 错误信息通过 tracer 记录

## 4. 集成方案

### 4.1 启动流程集成

MCP 的初始化发生在 GraphAgent 启动阶段，在 ToolCenter 完成 `auto_discover()` 之后：

```
GraphAgent 启动流程:
  1. ToolCenter().auto_discover()          ← 注册内置工具
  2. SkillRegister().load_all()            ← 注册脚本工具
  3. SubAgentRegistry(skill_register)      ← 构建 SubAgent 注册表
  4. MCPManager(tool_center).setup()       ← [新增] 注册 MCP 工具
  5. build_orchestration_graph()           ← 构建编排图
```

在代码层面的集成点——`subagent_exec_node` 所在的模块（`node/subagent.py`）中已有模块级的 `_tool_center` 和 `_registry` 单例。MCP 的初始化应在创建 `_registry` 之后、构建编排图之前完成。

推荐在 `orchestration/__init__.py` 或 `node/subagent.py` 顶部添加：

```python
_tool_center = ToolCenter()
_tool_center.auto_discover()

# [新增] 初始化 MCP 工具
_mcp_manager = MCPManager(_tool_center)
_mcp_manager.setup()
```

### 4.2 运行时调用流程

```
用户请求
    │
    ▼
GuardAgent (意图分析)
    │
    ▼
PlanAgent (任务分解)
    │ required_skill: "database_query"
    │
    ▼
SubAgent (subagent_exec_node)
    │ LLM 决定调用 mcp__sqlite__read_query
    │
    ▼
ToolCenter.tools["mcp__sqlite__read_query"].run(**args)
    │
    ▼
_call_mcp_tool("sqlite", "read_query", args)
    │
    ▼
session.call_tool("read_query", args)   ← MCP 异步调用
    │
    ▼
CallToolResult → AgentTool.return → ToolMessage
```

对上游组件（SubAgent LLM、PlanAgent、GuardAgent）而言，MCP 工具与内置工具完全一致——都需要 LLM 通过 tool_call 选择调用，ToolCenter 对调用来源不做区分。

### 4.3 工具命名规范

| 工具类型 | 命名格式 | 示例 |
|---------|---------|------|
| 内置工具 | `<name>` | `read_file`, `safe_calculator` |
| 脚本工具 | `<script_name>` | `describe_csv.py`, `step1_preprocess.py` |
| **MCP 工具** | `mcp__<server>__<tool>` | `mcp__sqlite__read_query`, `mcp__remote-api__get_user` |

命名设计理由：
- `mcp__` 前缀一眼可识别来源
- 双下划线 `__` 分隔 server 和 tool 名称，因为 server name 来自用户配置的人读名称（通常不含 `__`），tool name 来自 MCP Server 的声明（也不含 `__`）
- 若同一个 MCP Server 的 tool name 本身就包含 `__`（极罕见），不影响解析（只需从 `mcp__` 去除前缀，然后以 `__` 分割第一次得到 server，剩余部分为 tool）

### 4.4 与 Hook 系统的协作

MCP 工具注册时使用的是标准 `AgentTool` 类，因此天然继承 `AgentTool.run` 中的 Hook 检查点：

```
AgentTool.run(**kwargs)
    ├── before_tool_call Hook  ← Hook 可以拦截/修改 MCP 工具调用
    ├── self.func(**kwargs)   ← 实际执行 _call_mcp_tool
    └── after_tool_call Hook  ← Hook 可以观察 MCP 工具调用结果
```

不需要为 MCP 工具做任何额外的 Hook 适配。

## 5. 错误处理

### 5.1 错误分类与处理策略

| 错误类型 | 发生阶段 | 处理方式 |
|---------|---------|---------|
| `mcp_servers.json` 不存在 | 启动 | 跳过 MCP 初始化，GraphAgent 正常运行 |
| JSON 解析失败 | 启动 | 记录错误日志，跳过 MCP 初始化 |
| 配置格式无效（缺少必填字段） | 启动 | 跳过该 Server，记录错误，继续处理下一个 |
| MCP Server 进程启动失败（stdio） | 连接 | 跳过该 Server，记录错误，继续处理下一个 |
| HTTP 连接超时/被拒（streamable-http） | 连接 | 跳过该 Server，记录错误，继续处理下一个 |
| `initialize()` 失败 | 连接 | 跳过该 Server，记录错误 |
| `list_tools()` 失败 | 工具发现 | 跳过该 Server，记录错误 |
| 单个工具注册冲突（同名） | 工具注册 | 记录警告，跳过冲突工具 |
| `call_tool()` 超时 | 运行时 | 返回错误文本给 LLM，由 LLM 决定重试或回退 |
| `call_tool()` 返回 `isError=True` | 运行时 | 提取错误文本作为工具返回结果传给 LLM |
| Session 断开（子进程退出/HTTP 连接断开） | 运行时 | 返回错误文本给 LLM，后续调用同样返回错误 |

### 5.2 容错原则

- **启动时容错**：任何一个 MCP Server 的连接/发现失败不影响其他 Server，也不影响 GraphAgent 正常功能
- **运行时容错**：MCP 工具调用失败时，向 LLM 返回清晰的错误信息（包含 Server 名称、工具名、错误原因），LLM 可以根据错误信息决定下一步动作（重试、换用其他工具、或向用户报告）
- **无自动重连**：为避免引入连接重试的复杂性（退避策略、最大重试次数、状态恢复），初始版本不实现自动重连。session 断开后的调用一律返回错误，用户需重启 GraphAgent 恢复连接

## 8. 设计决策记录

| 决策点 | 选项 | 选择 | 理由 |
|--------|------|------|------|
| 工具命名前缀 | `mcp__` vs `mcp:` vs `mcp/` | `mcp__` | 双下划线在 JSON key 和函数名中天然合法，不需额外编码；`:` 在某些工具系统中需要转义；`/` 暗示路径层级 |
| 全局 session vs 依赖注入 | 全局 `_sessions` 字典 vs 闭包捕获 | 全局字典 | AgentTool 的 func 签名固定，无法在运行时传递 session 参数；全局字典是项目现有模式（`_registry`、`_tool_center` 均为模块级单例） |
| 同步/异步桥接 | asyncio.run vs 事件循环线程 | 按优先级组合 | 大多数情况（同步调用）直接 `asyncio.run()` 最简洁；少数情况（已在事件循环中）用子线程隔离 |
| 连接模式 | 长连接 vs 短连接 | 长连接 | stdio transport 必须保持子进程存活；streamable-http 保持连接池避免重复建连 |
| 自动重连 | 支持 vs 不支持 | 初始版本不支持 | 自动重连涉及退避策略、幂等性保证、状态恢复等复杂问题，先保持简单，后续按需添加 |
| 单个 Server 失败处理 | 全部失败 vs 跳过故障 Server | 跳过故障 Server | MCP 是可选增强，不应因单个外部服务不可用阻止 GraphAgent 启动 |
| 配置文件格式 | JSON vs YAML | JSON | `mcp_servers.json` 是 MCP 生态的事实标准（Claude Desktop、VS Code 等均使用此文件名和格式） |
